# 常规设置

**入口：** 侧边栏 → **设置** → **常规**

常规设置包含外观、本地化、数据存储、代理网络、环境变量等全局选项。每次修改后立即生效（数据目录迁移除外）。

![常规设置页面总览，左侧导航高亮"常规"，右侧依次展示主题、语言、数据目录等配置项](/images/general_1.png)

---

## 主题

**作用：** 更改整体界面配色方案，同时影响 Markdown 渲染与代码高亮颜色；切换即时生效，无需重启。

**默认值：** 深色 · 玄墨

共提供 3 套「宣纸墨金」主题，以卡片形式并排展示，点选即切换：

| 主题 | 风格简介 |
|------|---------|
| **深色 · 玄墨** | 玄墨深色底配淡金强调色（默认） |
| **浅色 · 宣纸** | 暖调宣纸白底配墨金强调色 |
| **护眼 · 竹青** | 浅竹青底配黛青强调色，适合长时间阅读 |

**跟随系统：** 主题卡片上方有「跟随系统」开关。开启后，界面会随操作系统的深色 / 浅色外观自动在**玄墨**与**宣纸**之间切换；**护眼 · 竹青**不参与自动跟随，需手动选择（选择护眼时会自动关闭「跟随系统」）。该开关默认关闭。

**操作：** 直接点选目标主题卡片即可切换；如需随系统自动切换深浅色，打开上方的「跟随系统」开关。

::: tip 旧主题自动迁移
早期版本提供的 8 套主题已收敛为上述 3 套「宣纸墨金」主题。升级后，你原有的主题选择会自动映射到最接近的新主题，无需手动重设。
:::

## 语言

**作用：** 切换整个应用界面的显示语言。

**可选值：** 中文、English

**操作：** 在下拉列表中选择语言；切换后界面立即刷新。

## 数据目录

**作用：** 所有任务记录、设置文件、日志等本地数据的存储根目录。

**默认路径：**
- Windows：`%APPDATA%\com.loongcode.desktop\`
- macOS：`~/Library/Application Support/com.loongcode.desktop/`

**操作：** 点击目录路径旁的文件夹图标选择新路径，确认后应用将自动把现有数据迁移至新位置。

::: warning 迁移前请备份
数据目录迁移会将全部数据复制到新路径并切换引导路径。迁移完成前请勿关闭应用，迁移失败时旧目录数据不会被删除，但新目录可能存在不完整文件。建议在迁移前手动备份旧目录。
:::

## CLI 命令

**作用：** 指定 Claude CLI 与 Codex CLI 可执行文件的路径或命令名称，用于在任务中调用对应 CLI。

应用会在系统 `PATH` 中自动发现 `claude` 与 `codex` 命令。若你将 CLI 安装在非标准位置或使用了自定义包装脚本，可在此手动填入绝对路径。

两项配置分别对应：

- **Claude CLI 命令** — 调用 Claude CLI 使用的命令或路径
- **Codex CLI 命令** — 调用 Codex CLI 使用的命令或路径

> **OpenCode CLI 无需在此手动配置**（v0.9.0 起）：应用会自动在系统 `PATH` 及 `~/.opencode/bin`、scoop shims 等常见位置发现 `opencode` 命令，因此本节不提供 OpenCode 的路径输入框。

**操作：** 在输入框中直接填写命令名（如 `claude`）或可执行文件的绝对路径（如 `/usr/local/bin/claude`）；也可点击旁边的"选择文件"按钮通过文件浏览器定位。

> 若未安装对应 CLI，请先参阅 [环境准备](/guide/prerequisites) 完成安装。

## 网络代理

**作用：** 配置应用发出的所有网络请求（包括 CLI 调用与依赖下载）所使用的 HTTP/HTTPS 代理。

| 字段 | 说明 |
|------|------|
| **网络代理** | 代理服务器地址，格式 `http://host:port` 或 `https://host:port` |
| **忽略代理的地址** | 不经过代理的地址列表，多个地址以逗号分隔（如 `localhost,127.0.0.1`） |
| **自定义 CA 证书文件** | 企业内网或自签名证书场景下，选择 `.pem` / `.crt` / `.cer` 格式的根证书文件 |

**操作：** 填写代理地址后按回车或点击其他区域保存；代理地址必须包含 `http://` 或 `https://` 前缀，否则会提示格式错误。

## 环境变量

**作用：** 为 CLI 进程注入额外的环境变量，例如 API Key、工具链路径等。此处配置的变量仅对应用启动的 CLI 子进程生效，不影响系统级环境。

**操作：** 点击"添加"按钮，输入变量名与变量值，回车或点击保存即可生效。

**示例：** 变量名填 `ANTHROPIC_API_KEY`、变量值填对应密钥，Claude CLI 即可读取该密钥访问 Anthropic 兼容接口（敏感凭据的注意事项见下方提示）。

::: warning 请勿在此存储高敏感凭据
环境变量明文保存在本地配置文件中。API Key 等敏感凭据建议通过各 CLI 自身的登录/认证流程管理，或使用操作系统的密钥链，而非在此处硬编码。
:::

## 提示音

**作用：** 任务完成或失败时播放短促的提示音，方便在后台运行时得到通知。

**默认值：** 开启

**操作：** 点击开关切换启用/禁用状态。

---

## 回合代码快照

**作用：** 在每轮对话开始前，自动对工作区代码文件拍摄 Git 快照（提交），以便在编辑历史消息时可一并回滚代码变更。

**默认值：** 开启

**依赖：** 需要系统中已安装 `git`。若 git 未安装，开关将无法生效。

**操作：** 点击开关切换启用/禁用状态。

> 关于快照的查看与回滚操作，请参阅 [代码快照](/features/checkpoints)。

---

## 自动压缩

**作用：** 当对话上下文接近模型上下文窗口上限时，自动执行 `/compact`（压缩摘要）并继续任务，避免因超限导致任务中断。超限产生的错误会被隐藏，仅保留一条系统提示。

**默认值：** 开启

**操作：** 点击开关切换启用/禁用状态。

---

## 自动归档

**作用：** 自动归档超过指定时长未被激活的任务（按最近激活时间计算）。以下任务不受自动归档影响：
- 正在运行的任务
- 当前正在查看的任务
- 已手动置顶的任务

**默认值：** 开启，阈值 7 天

**配置项：**

- **启用开关** — 打开/关闭整个自动归档功能
- **阈值数字** — 多长时间未激活触发归档（最小值为 1）
- **时间单位** — 小时 / 天 / 月

**操作：** 开启开关后，设置阈值数字与单位即可；调整后自动保存并立即按新规则重新扫描。

> 关于任务归档与工作区管理，请参阅 [任务工作区](/features/tasks-and-workspaces)。

## 使用引导

**作用：** 分章重看首次启动时自动播放的新手引导，帮助你重新熟悉界面布局与核心功能入口。

该区块右上角提供**从头重看**按钮，可重新播放完整引导；下方按章节列出全部 5 章（起步、看代码、进阶选项、Git、设置），每章右侧有**重看**按钮可单独重放，已完成的章节会显示绿色对勾。

> 新手引导的完整内容介绍见 [快速上手](/guide/quick-start)。
