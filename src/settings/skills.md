# 技能

**入口：** 侧边栏 → **设置** → **技能**

技能（Skill）是预置的专项工作流，本质上是存储在特定路径的 Markdown 提示词文件。在对话中可通过 `/` 命令面板按名称调用，由模型按照技能文件中的指引执行专项任务。

本页管理各 CLI 各自的技能列表，通过顶部 **Claude | Codex | OpenCode** 切换栏分别维护（OpenCode 家族自 v0.9.0 起纳入）。

> Claude 模式下技能通过 `/` 命令面板调用，详见 [斜杠命令](/features/slash-commands)；Codex 模式下技能文件由 Codex CLI 直接加载，模型会根据技能的名称与描述自动判断是否调用，也可在 `/` 命令面板中显式选中；OpenCode 模式的技能来自与 Claude 共享的技能目录及 OpenCode 自有目录，启用开关控制其对 OpenCode 是否生效。

## 技能列表

列表中每条技能记录显示：

- **名称** — 技能标识，即在 `/` 面板中输入的名称
- **作用域标签** — 蓝色 **用户**、绿色 **工作区** 或紫色 **Plugin**（来自插件提供的内置技能）
- **描述** — 技能用途的一句话说明
- **启用开关**（Claude / OpenCode 模式下显示）— 即时启用或禁用；禁用后该技能从 `/` 面板中隐藏

点击列表中的任意一行可打开 **技能详情** 弹窗，查看技能的名称、描述、作用域、状态和文件路径。

---

## 作用域说明

| 作用域 | 存储位置 | 说明 |
|--------|---------|------|
| **用户** | 用户级配置目录 | 跨工作区生效 |
| **工作区** | 工作区目录下 | 仅对当前工作区生效 |
| **Plugin**（界面原文） | 由插件附带 | 来自已安装插件，附加 **项目** 或 **用户** 子作用域标签 |

---

## 创建技能

### Claude 模式

点击列表顶部的 **创建技能** 按钮：

- 若已关联工作区任务，将自动在对话面板中启动一个新任务，并调用 `skill-creator` 技能引导你逐步填写技能内容
- 若未关联工作区任务，按钮不可点击，悬停提示"需先选中某个工作区的任务"

### Codex 模式

点击 **创建技能** 按钮后弹出 **新建 Codex 技能** 对话框，填写以下字段后点击 **创建**：

| 字段 | 说明 |
|------|------|
| **名称** | 技能标识，如 `pdf-processing` |
| **描述** | 一句话说明技能用途与触发时机 |

### OpenCode 模式

OpenCode 模式**不提供新建入口**（v0.9.0 起）——它天生与 Claude 共享技能目录，因此本页只**列出**这些技能并提供启用开关，不重复创建。若要新增技能，在 **Claude** 模式下创建即可，OpenCode 会自动共享。

---

## 在对话中使用技能

Claude 模式下，在输入框输入 `/` 打开命令面板，技能会与插件命令一同列出，带有琥珀色的 `skill` 类型标识。输入技能名称可快速筛选并选中。

Codex 模式下无需手动触发：技能文件存放在 Codex CLI 识别的目录中（用户级 `~/.codex/skills/`，项目级 `<工作区>/.codex/skills/`，以及已安装插件附带的技能），由 Codex CLI 直接加载，模型根据每个技能的名称与描述自动判断是否使用；你也可以在 `/` 命令面板中显式选中某个技能随消息发送。Codex 模式没有逐技能的启用开关。

OpenCode 模式下，技能来自与 Claude 共享的加载根及 OpenCode 自有目录。本页的启用开关会写入应用托管的 OpenCode 配置（`permission.skill`，并保留其中的供应商 / MCP 段），控制每个技能对 OpenCode 是否生效；已启用的技能也会显示在 `/` 命令面板中。
