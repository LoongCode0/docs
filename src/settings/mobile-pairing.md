# 移动端配对（LoongCode App）

**入口：** 设置 → **移动端配对**（侧边栏手机图标）

移动端配对（**v0.8.0 起**）让你把官方的 **LoongCode App**（手机端伴侣应用）与本机电脑绑定，离开电脑时也能用手机远程查看与驱动电脑上的 AI 任务。它通过一个**中转服务**在手机与电脑之间转发消息，所有内容**端到端加密**——中转服务看不到明文。

> 电脑始终是「大脑」：CLI 子进程、会话历史与项目文件都留在电脑上，手机只是远程的输入与查看端。

::: warning 安全提醒：配对即授予操作权限
完成配对的手机可以远程让电脑上的 AI **修改、删除文件并运行命令**。请只与你本人持有的手机配对，妥善保管配对二维码，并在不再使用某台设备时及时**解绑**。
:::

::: tip 与「机器人渠道」的区别
[机器人渠道](/settings/bots)（微信 / 飞书）是把即时通讯消息转发给 AI，面向纯文本交互；**移动端配对**则是专用的 LoongCode App，支持实时事件流、历史回放、富输入（斜杠命令 / @文件 / 图片）与任务配置同步等更完整的能力。两者相互独立，可按需选用。
:::

---

## 工作原理

1. 电脑（PC 端）通过一个**中转服务**接入网络，并生成配对二维码。
2. 手机上的 LoongCode App 扫码完成配对，与电脑建立端到端加密通道。
3. 之后手机即可远程浏览电脑的工作区与任务、打开任务回放历史、发送消息并实时看到 AI 的流式回复。

中转服务只负责盲转发加密数据，无法读取消息内容；密钥与凭据只保存在电脑本地，前端界面只展示绑定状态与时间戳。

## 配置步骤

移动端配对页从上到下分为三个区块：**中转服务器配置**、**配对二维码**、**已配对设备**。

### 1. 中转服务器配置

| 字段 | 说明 |
|------|------|
| **中转地址** | **留空**即使用 LoongCode 官方公共中转，开箱即用；如需自建，可填入私有部署的 `ws://` / `wss://` 地址 |
| **令牌** | 使用官方公共中转时**留空**；私有部署若设置了访问令牌则在此填写 |

填好后点击 **保存并连接中转**。保存成功后下方会显示连接状态：

- **已连接中转站**（绿色）：与中转服务连接正常。
- **中转已保存，正在连接…**：正在建立连接，稍候即可。

::: tip
绝大多数用户无需填写任何地址与令牌——直接留空保存即可走官方公共中转。中转地址与令牌仅在私有部署场景下才需要填写。
:::

### 2. 配对二维码

中转连接就绪后，**生成配对二维码** 按钮才可用。点击后页面显示一个二维码：

1. 打开手机上的 **LoongCode App**。
2. 用 App 扫描电脑屏幕上的二维码。
3. App 提示配对成功后，该设备会出现在下方的「已配对设备」列表中。

二维码内含一次性配对码（短时有效），用于完成密钥协商；过期后重新点击**生成配对二维码**即可刷新。

### 3. 已配对设备

列出当前已绑定的所有设备，每条显示设备名称与配对日期：

- 点击右上角 **刷新** 重新拉取列表。
- 点击设备行末的 **解绑** 即可移除该设备；解绑后该设备需重新扫码才能再次使用。

## 在手机上能做什么

完成配对后，LoongCode App 可以远程操作电脑上的任务：

- **浏览工作区与任务**：查看电脑上的真实工作区列表，以及各工作区下任务的实时状态（空闲 / 运行中 / 提问中）。
- **打开任务并回放历史**：打开一条既有任务即加载最近的对话历史，上滑可分页加载更早的消息，用户气泡 / 助手气泡 / 工具调用气泡分别呈现。
- **新建任务**：直接从手机发起新任务，标题自动取首条消息摘要。
- **实时收发消息**：在手机上发送的消息会注入电脑端的 AI 会话，AI 的流式回复实时同步到手机。无论消息从手机还是电脑发出，两端都保持一致。
- **富输入**：行首输入 `/` 调出**斜杠命令**面板（含电脑端真实的命令 / 技能 / 工作流）、输入 `@关键字` **搜索并引用工作区文件**、以及从相册或拍照**发送图片**。
- **回答提问与权限审批**：AI 抛出的提问（AskUserQuestion）与命令 / 改文件的权限请求会同步到手机，可在手机上直接作答或批准 / 拒绝，结果实时回灌到电脑端。
- **任务配置同步**：在手机上调整任务的 CLI、模型、推理强度与权限预设（「调校四件套」），改动落到电脑端并在下一轮生效；电脑端在桌面 GUI 上的同类改动也会实时同步回手机。

::: tip 本地任务隔离
你在**电脑上**直接新建的任务，默认不会主动推送到手机；只有在手机上**打开**该任务后，电脑才开始向手机推送它的实时事件。关闭任务对话即停止推送。
:::

## 连接稳定性

- 网络抖动时，电脑会自动重连中转服务并补发未确认的消息，短暂断网不丢消息。
- 电脑与中转之间有应用层心跳与空闲检测，能及时发现并修复「假在线」状态。
- 临时网络波动**不会清除配对凭据**；只有明确的鉴权失效才需要重新配对。

---

## 已知限制

- 当前为初期版本，更多电脑端能力（如终端、文件、Git 的远程操作）仍在后续规划中。
- 富输入中的 `@文件` 在 worktree 任务下以工作区根目录为基准解析相对路径。
- 所有模型交互仍由电脑本机的 CLI 子进程（Claude Code / Codex）完成，中转服务与 LoongCode 均不直接访问模型 API。
