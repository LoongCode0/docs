依赖管理 ​

入口： 侧边栏 → 设置 → 依赖管理

依赖管理页面用于检测与安装应用所需的外部工具（CLI 及运行时）。相比 环境准备 中的首次安装引导，本页提供更完整的依赖状态查看与安装来源配置。

依赖分组 ​

依赖按重要性分为两组：

分组说明
必须依赖核心功能所需，缺少时任务无法运行（如 Claude CLI）
可选依赖扩展功能所需，缺少时部分功能受限（如 Codex CLI、git）

依赖状态检测 ​

页面加载时自动扫描系统环境，每个依赖项显示以下信息：

  • 状态： 已安装的依赖显示绿色已安装标签；未安装则显示安装入口
  • 来源标签： 应用安装 / 系统安装 / 未安装
  • 版本号： 已安装时显示检测到的具体版本
  • 操作按钮（仅未安装时显示）： 支持应用内安装的依赖显示安装按钮（点击后变为安装中…）；不支持应用内安装的依赖显示触发系统安装按钮，引导通过系统方式完成安装

安装依赖 ​

点击依赖项右侧的安装按钮后，进度日志会实时展示在下方：

日志阶段含义
下载中 xx%正在从指定来源下载安装包
解压中…正在解压下载的文件
执行中…正在运行安装脚本
失败：安装失败，错误详情保留在日志区域

安装日志在关闭对话框后仍然保留，导航到其他页面再返回依然可以查看。

安装完成后需重启应用

新安装的工具需要重启应用才能被正确检测到，因为 PATH 环境变量在应用启动时已固定。请根据提示重启。

安装失败时，日志区域提供重试按钮。若反复失败，请参阅 故障排查 查看常见原因与解决方法。

安装来源配置 ​

入口： 依赖管理页面 → 安装来源 区域

安装来源决定应用从哪里下载 CLI 工具。提供三种模式：

来源说明适用场景
官方脚本使用各 CLI 官方提供的安装脚本网络直连、访问官方源无障碍时
GitHub 直连直接从 GitHub Releases 下载网络可访问 GitHub 时
GitHub 加速通过镜像服务转发 GitHub Releases国内网络访问 GitHub 受限时（推荐）

选择 GitHub 加速 时，还可在下方出现的输入框中填写自定义镜像前缀（v0.6.5 起支持），格式为 https://your-mirror.example.com/，留空则使用内置默认镜像。

国内网络建议

若你在中国大陆使用，GitHub 直连可能速度较慢或不稳定，建议选择 GitHub 加速 并使用内置默认镜像或自填可信镜像地址。

下载代理配置 ​

安装依赖时的下载请求支持独立的代理设置：

模式说明
跟随全局使用 常规设置 → 网络代理 中配置的代理
自定义单独为依赖下载指定代理地址
直连不使用任何代理，直接连接目标地址
