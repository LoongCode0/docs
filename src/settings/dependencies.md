# 依赖管理

**入口：** 侧边栏 → **设置** → **依赖管理**

依赖管理页面用于检测与安装应用所需的外部工具（CLI 及运行时）。相比 [环境准备](/guide/prerequisites) 中的首次安装引导，本页提供更完整的依赖状态查看与安装来源配置。

---

## 依赖分组

依赖按重要性分组显示：

| 分组 | 说明 |
|------|------|
| **AI CLI · 至少安装一个** | 三个 AI CLI 家族 **Claude / Codex / OpenCode** 归为一组（v0.9.0 起），任务至少需要其中一个才能运行；三者全部未安装时，分组顶部显示醒目提示「请至少安装一个:Claude / Codex / OpenCode」 |
| **必须依赖** | 其他核心工具，如 git |
| **可选依赖** | 扩展功能所需，缺少时部分功能受限 |

::: tip 三个 AI CLI 都是可选其一
你不必装齐三个 CLI——按需选用即可。例如只用 Claude、或只用 OpenCode 配合第三方供应商，都能正常发起任务。只要三者中已安装任意一个，分组警告即消失。
:::

## 依赖状态检测

![依赖管理页面，列出 Claude CLI、Codex CLI、OpenCode CLI、git 等依赖项，每项显示来源标签与版本号，未安装项右侧提供安装按钮](/images/dependencies_1.png)

页面加载时自动扫描系统环境，每个依赖项显示以下信息：

- **状态：** 已安装的依赖显示绿色**已安装**标签；未安装则显示安装入口
- **来源标签：** 应用安装 / 系统安装 / 未安装
- **版本号：** 已安装时显示检测到的具体版本
- **操作按钮（仅未安装时显示）：** 支持应用内安装的依赖显示**安装**按钮（点击后变为**安装中…**）；不支持应用内安装的依赖显示**触发系统安装**按钮——点击后调起操作系统自带的安装流程（目前对应 macOS 上的 git：弹出系统的命令行开发者工具安装对话框），完成后回到本页重新检测即可

## 安装依赖

点击依赖项右侧的**安装**按钮后，进度日志会实时展示在下方：

| 日志阶段 | 含义 |
|---------|------|
| 下载中 xx% | 正在从指定来源下载安装包 |
| 解压中… | 正在解压下载的文件 |
| 执行中… | 正在运行安装脚本 |
| 失败：{错误信息} | 安装失败，错误详情保留在日志区域 |

安装日志在关闭对话框后仍然保留，导航到其他页面再返回依然可以查看。

::: tip 安装完成后需重启应用
新安装的工具需要重启应用才能被正确检测到，因为 `PATH` 环境变量在应用启动时已固定。请根据提示重启。
:::

安装失败时，日志区域提供**重试**按钮。若反复失败，请参阅 [故障排查](/reference/troubleshooting) 查看常见原因与解决方法。

## 安装来源配置

**入口：** 依赖管理页面 → **安装来源** 区域

安装来源决定应用从哪里下载 CLI 工具。提供三种模式：

| 来源 | 说明 | 适用场景 |
|------|------|---------|
| **官方脚本** | 使用各 CLI 官方提供的安装脚本 | 网络直连、访问官方源无障碍时 |
| **GitHub 直连** | 直接从 GitHub Releases 下载 | 网络可访问 GitHub 时 |
| **GitHub 加速** | 通过镜像服务转发 GitHub Releases | 国内网络访问 GitHub 受限时（推荐） |

选择 **GitHub 加速** 时，还可在下方出现的输入框中填写**自定义镜像前缀**（v0.6.5 起支持），格式为 `https://your-mirror.example.com/`，留空则使用内置默认镜像。

::: tip 国内网络建议
若你在中国大陆使用，GitHub 直连可能速度较慢或不稳定，建议选择 **GitHub 加速** 并使用内置默认镜像或自填可信镜像地址。
:::

::: tip OpenCode CLI 的安装方式
**OpenCode CLI**（v0.9.0 起可在本页一键安装）在 Windows 上由应用自管从 GitHub 下载并解压安装（同样支持上述 GitHub 直连 / 加速来源）；在 macOS / Linux 上则调用官方安装脚本（`opencode.ai/install`）。安装完成后同样需按提示重启应用。
:::

## 下载代理配置

安装依赖时的下载请求支持独立的代理设置：

| 模式 | 说明 |
|------|------|
| **跟随全局** | 使用 [常规设置 → 网络代理](/settings/general#网络代理) 中配置的代理 |
| **自定义** | 单独为依赖下载指定代理地址 |
| **直连** | 不使用任何代理，直接连接目标地址 |
