# 任务与工作区

LoongCode 用**工作区**（Workspace，对应你磁盘上的项目目录）和**任务**（Task，对应一次 AI 对话会话）两级结构来组织你的工作。本页介绍如何添加工作区、管理任务生命周期，以及置顶、搜索、归档等高效操作。

> 如果你还没开始第一个任务，建议先看[快速上手](/guide/quick-start)了解基础操作流程。

## 左侧任务栏布局

左侧任务栏从上到下的真实元素顺序为：

| 区域 | 说明 |
|------|------|
| **Logo / 折叠按钮** | 悬浮后可折叠侧栏（v0.7.0 起） |
| **搜索框** | 按关键词实时筛选任务 |
| **＋ 打开工作区** | 添加项目文件夹为工作区 |
| **＋ 新建任务** | 快速新建一个任务草稿 |
| **已置顶** | 已置顶任务集合，独立于分组折叠 |
| **任务大组头** | 含「任务 / 归档」标签与视图切换（分组列表 / 时间线） |
| **工作区树 + 任务列表** | 按工作区分组，展示各任务 |
| **底部栏** | 新手引导、移动端支持、打开设置 |

::: tip 快捷键
- `Ctrl+N`：新建任务草稿（任意界面可用）
- 悬浮 Logo 点击：折叠 / 展开左侧任务栏（v0.7.0）
:::

## 添加与管理工作区

### 添加工作区

1. 点击搜索框下方的**＋ 打开工作区**按钮。
2. 在弹出的文件夹选取对话框中选择本地项目目录，点击**确认**。
3. 如果该目录下有 Claude Code 历史会话，会弹出询问「**同步会话**」——点**导入**可将历史记录导入为任务；点**跳过**则跳过。

工作区添加成功后会出现在任务列表树中。

### 工作区操作

鼠标悬浮到工作区名称行，右侧出现三个快捷图标：

- **新建任务**（`shell.newTaskForWs`）：在该工作区下新建任务
- **查看文件**：跳转到右侧面板的文件树视图
- **移除工作区**：从 LoongCode 中移除该工作区条目（**不影响磁盘上的实际文件**）

::: warning 移除工作区
移除工作区会删除该工作区下的所有任务记录（`shell.removeWorkspaceDesc`：界面原文如此），但磁盘上的代码与历史文件不受影响。
:::

## 新建任务

### 草稿态机制

点击**＋ 新建任务**（或按 `Ctrl+N`）后，任务进入**草稿态**——此时尚未写入数据库，主区域内联显示配置选择器供你挑选工作区、模型、Git 分支或 worktree（Git worktree 是 Git 的独立检出目录，允许同时在多个分支上工作）。

**发送第一条消息后，任务才正式创建**，这样可以避免空任务堆积。

草稿会默认选中上次用过的工作区，少点一步。草稿选中的工作区被外部删除时，发送按钮会被禁用并给出提示。

### 草稿态下可用功能

即使任务还在草稿态，右侧面板的文件树、编辑器、审查、浏览器、终端均可正常使用，它们按工作区工作，不依赖任务是否已落盘。

## 任务状态

每个任务行左侧的彩色圆点表示当前状态，共五种：

| 状态 | 含义 |
|------|------|
| **空闲** | 无正在运行的回合 |
| **运行中** | AI 正在生成回复或执行工具 |
| **成功** | 最近一轮回合正常结束 |
| **出错** | 最近一轮回合以错误结束 |
| **提问中** | AI 正在等你回答一个问题，任务行蓝色高亮 |

## 任务排序与置顶

### 按最近活动排序

任务列表与时间线分桶按**最近活动时间**排序——刚发过消息或刚结束一轮对话的任务会自动浮到前面（v0.4.2 起）。

### 置顶任务

对于常用任务，可以将其置顶：

1. 右键点击任务行（或点击行尾的操作菜单）。
2. 选择**置顶任务**（界面文案：`shell.pinTask`）。

置顶后任务出现在**已置顶**分组，跨视图、不受「折叠全部」影响。取消置顶选**取消置顶**（`shell.unpinTask`）即可。

::: tip
置顶任务受自动归档保护，不会被自动归档。
:::

## 搜索任务

搜索框（输入框占位符文案：`composer.searchTaskPlaceholder` 「搜索任务…」）位于侧栏顶部 Logo 区域下方，输入关键词后任务列表实时筛选。

- 无匹配时显示：「未找到匹配任务」（`shell.noMatchedTasks`）
- 点击搜索框右侧的「清除搜索」按钮（`composer.searchClear`）可清空关键词

## 归档任务

### 手动归档

右键点击任务行，选择**归档**（`shell.archiveTask`）；也可以在右键菜单中点**归档**（`menu.archive`）。

归档后任务从主列表移出，进入归档视图，主列表更加清爽。

取消归档：进入归档视图后，右键任务选**取消归档**（`shell.unarchiveTask` / `menu.unarchive`），任务的活动时间会被重置，回到主列表最上方，方便立即继续。

### 自动归档

应用**启动时**与**每小时**各跑一次自动归档，按「最近活动时间」超过阈值自动将久未触碰的任务归档（v0.4.3 起）。

**保护规则**：运行中任务和已置顶任务不会被自动归档。

**调整阈值**：在**设置 → 常规**中可开关自动归档，并设置阈值数值与单位（小时 / 天 / 月），默认开启、7 天。

### 查看归档视图

1. 在任务大组头右侧点击**查看归档**（`shell.viewArchive`）。
2. 进入归档视图后，任务按归档时间降序排列。
3. 归档视图下自动隐藏置顶区和新建入口，空态有友好提示：「暂无归档任务」（`shell.noArchivedTasks`）。
4. 点击**退出归档**（`shell.exitArchive`）返回主任务列表。

## 任务右键菜单

右键任务行可弹出包含以下操作的菜单（文案来自 shell.ts / contextMenu.ts）：

| 操作 | 界面文案 |
|------|---------|
| 重命名 | **重命名**（`shell.renameTask`） |
| 置顶 / 取消置顶 | **置顶任务** / **取消置顶**（`shell.pinTask` / `shell.unpinTask`） |
| 归档 / 取消归档 | **归档** / **取消归档**（`shell.archiveTask` / `shell.unarchiveTask`） |
| 删除 | **删除**（`shell.deleteTask`） |

::: danger 删除任务
删除操作不可恢复（界面确认弹窗文案：`shell.deleteTaskDesc`）。删除任务仅删除 LoongCode 内的任务记录，**不影响磁盘上的历史 `.jsonl` 文件**。
:::

## 视图模式

任务大组头提供两种视图切换（`shell.filterView`）：

- **分组列表**（`shell.viewGrouped`）：按工作区分组展示，可展开 / 折叠各工作区
- **时间线**（`shell.viewTimeline`）：按时间桶（今天、昨天、本周……）排列，跨工作区浏览

---

## 未读提示

任务行右侧显示角标，表示该任务有未读的新消息（例如 AI 在后台完成了回复，而你正在查看其他任务）。切换到该任务查看消息后，角标自动消除。
