# CLI 与任务配置

**入口：** 任务输入栏底部选择器栏（Selector Bar）

任务级配置控制单个任务使用哪套 CLI 工具、哪个模型、多深的思考深度以及多宽松的执行授权。每个任务的配置相互独立。

**默认值的行为：** 首次新建任务时使用出厂默认（各项默认值见下文）；当你修改过任意任务的配置后，整套配置会被记为「最后配置」，之后新建任务会自动继承你上次的选择（v0.4.0 起），无需每次重新调整。

> 若尚未安装 Claude CLI 或 Codex CLI，请先参阅 [环境准备](/guide/prerequisites)。CLI 的可执行文件路径可在 [常规设置 → CLI 命令](/settings/general#cli-命令) 中自定义。

---

## 双 CLI 架构

自 v0.7.0 起，应用同时支持 **Claude CLI** 和 **Codex CLI** 两套工具链，可在同一工作区中混合使用：

| 工具 | 标识 | 适用场景 |
|------|------|---------|
| Claude CLI | Anthropic 图标 | Claude 系列模型，支持 OAuth 直连或第三方供应商 |
| Codex CLI | OpenAI 图标 | Codex/GPT 系列模型，支持 OpenAI 兼容接口 |

在任务的选择器栏中点击 CLI 图标或名称，即可在两者之间切换。

## 任务级配置项

![任务输入栏底部选择器栏，依次展示 CLI 选择、模型档位、思考深度、执行授权四组控件](...)

### CLI 选择

选择本任务使用的工具链：**Claude CLI** 或 **Codex CLI**。选择后，下方的模型、思考深度、执行授权选项会随之切换为对应工具链的选项。

---

### 模型

选择本任务实际调用的模型档位。

**Claude CLI 模型档位**（由轻量到强力）：

| 档位 | 对应角色 |
|------|---------|
| Haiku | 轻量任务 |
| Sonnet | 常规任务 |
| Opus | 复杂任务 |
| Fable | 专项档位（v0.6.5 起） |

**出厂默认：** Opus

> 各档位对应的具体模型版本由当前激活的[模型供应商](/settings/providers)决定。

**Codex CLI 模型：** 通过 [模型供应商](/settings/providers) 中的 Codex OAuth 或自定义供应商配置；默认使用 `gpt-5.5`。

---

### 思考深度

控制模型在回答前花多少算力进行内部推理（即"扩展思考"功能的深度）。

**Claude CLI 思考深度**（由浅到深）：

| 档位 | 界面原文 |
|------|---------|
| 低 | low |
| 中 | medium |
| 高 | high |
| 超高 | xhigh |
| 最高 | max |
| 极限 | ultracode |

**Codex CLI 思考深度**（由浅到深）：

| 档位 | 界面原文 |
|------|---------|
| 极简 | minimal |
| 低 | low |
| 中 | medium |
| 高 | high |
| 超高 | xhigh |

**出厂默认：**
- Claude CLI：max
- Codex CLI：medium

思考深度越高，回答质量通常越好，但耗时与 Token 消耗也相应增加。若希望加快响应，可手动调低档位。

---

### 执行授权

控制 CLI 在执行文件操作、运行命令等行为时的自主程度。

**Claude CLI 授权模式：**

| 模式 | 说明 |
|------|------|
| 默认（default） | 每次执行前询问用户确认 |
| 计划模式（plan） | 先生成完整操作计划，确认后再执行 |
| 自动编辑（acceptEdits） | 文件编辑操作自动通过，其他操作仍需确认 |
| 完全自主（bypassPermissions） | 所有操作自动通过，无需确认 |

**Codex CLI 授权模式：**

| 模式 | 说明 |
|------|------|
| 只读 | 仅允许读取文件，不执行写入或命令 |
| 自动 | 自动执行大多数操作，高风险操作询问确认 |
| 完全访问 | 所有操作自动通过 |

**出厂默认：**
- Claude CLI：完全自主（bypassPermissions）
- Codex CLI：自动

::: warning 出厂默认为完全自主
Claude CLI 的出厂默认即为"完全自主"——该模式与 Codex 的"完全访问"会跳过所有确认步骤，CLI 可在无人值守情况下修改、删除文件或执行系统命令。若你希望逐步确认每个操作，请改为"默认"或"计划模式"（修改后新建任务会沿用你的选择）。建议仅在受控环境（沙盒、CI）或你完全了解任务内容时使用完全自主模式。
:::

## 与全局设置的关系

任务级配置仅影响当前任务，不修改全局默认值。关系如下：

| 配置层级 | 位置 | 作用范围 |
|---------|------|---------|
| CLI 可执行文件路径 | [常规设置 → CLI 命令](/settings/general#cli-命令) | 全局，所有任务共用 |
| 模型供应商 / API Key | [模型供应商](/settings/providers) | 全局，决定可用的模型集合 |
| 模型 / 思考深度 / 授权 | 任务选择器栏 | 单任务，新建任务继承上次配置 |
