# 分享设置

**入口：** 设置 → **分享**

配置[会话分享](/features/session-share)所使用的分享服务器（v0.9.8 起）。**绝大多数用户无需任何配置**——两项留空即使用 LoongCode 官方分享服务，开箱即用；仅当团队自行部署了分享服务器（例如出于内网合规需要）时才需要填写。

---

## 配置项

| 字段 | 说明 |
|------|------|
| **服务器地址** | **留空**即使用官方分享服务器；自部署时填写你的分享服务根地址（`https://` 开头） |
| **访问密钥** | **留空**即使用内置密钥（配合官方服务）；自部署时填写与你的服务器端一致的访问密钥 |

修改后点击**保存**生效。

::: tip 密钥的作用
访问密钥用于对创建 / 撤销分享的请求做签名校验，防止分享服务被无关客户端滥用；它不加密分享内容本身。分享页的访问控制依靠链接中的随机段与有效期；v0.9.10 起还支持可选的**密码保护**（创建时由客户端生成随机密码附在链接中，服务器先验密码再返回内容），详见[会话分享 → 隐私与安全](/features/session-share#隐私与安全)。
:::

::: warning 自部署注意
- 服务器地址与访问密钥必须与你部署的分享服务端配置匹配，密钥不一致时创建分享会提示「访问密钥无效，请检查设置」。
- 更换服务器地址只影响**之后**的操作；此前在旧服务器上创建的分享链接仍托管在旧服务器上。若需让旧链接提前失效，请在**切换配置前**先完成撤销，或等待其自然过期。
:::
