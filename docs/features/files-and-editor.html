文件与编辑器 ​

LoongCode 内置了一套完整的文件管理与编辑能力：左侧栏可切换为文件树浏览项目目录，右侧面板可打开文件查看或修改、对比前后差异，图片、Office 文档、表格数据、音视频等几十种格式也都能就地预览……无需切换到外部编辑器或应用，一切在应用内完成。

相关基础：界面导览

文件树 ​

文件树是左侧栏的一种视图（与任务列表视图切换），展示当前工作区的完整目录结构，并实时监听磁盘变动——AI 新建或删除文件时，文件树会自动刷新，无需手动操作。

打开文件树 ​

在左侧任务栏中，鼠标悬浮到工作区名称行，右侧出现快捷图标，点击查看文件即可将左侧栏切换为该工作区的文件树视图。

点击文件树顶部的返回任务按钮，退回任务列表视图。

新建文件与文件夹 ​

新建操作通过右键菜单完成：

  1. 在文件树的空白处右键，选择新建文件或新建文件夹，新条目落在工作区根目录；也可以右键某个文件或文件夹，在当前目录下新建（见下文「右键菜单」）。
  2. 弹出输入对话框，输入文件名（或文件夹名）后点击创建即完成。

TIP

文件名不能包含路径分隔符，也不能为 . 或 ..。

右键菜单 ​

在文件树中右键单击文件或文件夹，弹出快捷菜单，包含以下操作：

菜单项快捷键说明
复制Ctrl+C复制选中文件到系统剪贴板
剪切Ctrl+X剪切选中文件到系统剪贴板
粘贴Ctrl+V粘贴剪贴板中的文件（剪贴板没有文件时该项置灰）
重命名F2将文件或文件夹改名
删除Del删除选中文件或文件夹
在资源管理器中查看—在系统资源管理器中定位并高亮该文件
复制绝对路径—将文件的完整绝对路径复制到剪贴板
复制相对路径—将相对工作区根目录的路径复制到剪贴板
新建文件—在当前目录下新建文件
新建文件夹—在当前目录下新建文件夹

TIP

除 F2 外（F2 目前需通过右键菜单触发重命名），表中其余带快捷键的操作不必打开菜单——在文件树中选中条目后直接按对应按键即可，支持多选后批量复制、剪切或删除。

编辑器（文件标签） ​

右侧面板的文件标签集成了 Monaco（与 VS Code 同款的代码编辑内核），支持语法高亮与 Diff 对比视图；代码以外的图片、Office 文档、表格数据、音视频等格式则会自动路由到对应的专属预览器，详见下文文件预览。编辑器配色会跟随当前应用主题（深色 / 浅色 / 护眼）自动切换（v0.9.1 起）。每个打开的文件对应一个独立的标签，标题为文件名；有未保存修改时标题旁会显示脏点。

还没有打开任何文件时，右侧面板显示空状态提示「还没有打开的文件 · 点击 + 或按 Ctrl+P 打开」——点击标签栏右侧的 + 按钮，可选择打开文件（Ctrl+P）、审查或浏览器来新建标签。

打开文件 ​

方式一：在文件树中单击文件，文件在右侧面板新建文件标签打开。

方式二：Ctrl+P 文件搜索面板——在任意位置按 Ctrl+P，弹出搜索面板，输入文件名关键词后按回车即可打开目标文件。

方式三：从输入框内打开文件——在对话输入框输入 @ 选择文件，详见下文「在对话中 @ 引用文件」一节。

方式四：拖拽文件（v0.3.2 起）——从系统资源管理器拖任意文件到右侧面板、文件树或输入框，即可在右侧面板新建文件标签打开并编辑，支持工作区以外的文件。工作区以外的文件以面包屑形式（逐级目录路径）显示其完整的绝对路径，其标签页左侧色条与文件名会以警示色显示，提醒你它不属于当前工作区。

保存文件 ​

编辑完成后按 Ctrl+S 保存当前文件。若关闭标签页时有未保存的更改，会弹出确认对话框，提示「要保存对 [文件名] 的修改吗？」——选不保存直接关闭，否则先保存再关闭。

Diff 对比视图 ​

AI 修改文件时，对话区的工具卡片会内联显示 Diff（新旧代码行级对比，红绿高亮）。如需查看完整的代码变更，请切换到右侧面板的审查标签（详见 Git 集成 → 审查面板）。

Markdown 预览 ​

打开 .md 文件后，文件标签右上角出现切换到预览按钮，点击即可渲染 Markdown 预览；点击切换到编辑返回源码视图。Markdown、CSV、SVG、Jupyter Notebook、JSONL 这几种文本格式都共享这套「默认预览、可切源码」的双模式机制，具体渲染效果见下文文件预览。

文件预览 ​

除了 Monaco 源码编辑，LoongCode 会按扩展名自动把文件路由到合适的专属预览器打开——多数格式默认以预览态展示，无需先看一遍源码。标签栏右上角的在文件夹中显示按钮对任意文件都可用，可在系统文件管理器中定位并高亮该文件；Office 文档（docx / xlsx / xls / xlsm / pptx / pdf / doc / ppt）额外常驻一个用默认应用打开按钮，调用操作系统关联的默认程序打开。其余格式在预览失败或格式不受支持时，也会在错误提示页里补上这两个按钮作为出路（见下文预览失败时）。

图片 / 音频 / 视频 ​

  • 图片：png / jpg / jpeg / gif / webp / bmp / avif / ico
  • 音频：mp3 / wav / ogg / oga / flac / m4a / aac / opus / wma，内置播放控件
  • 视频：mp4 / webm / mov / mkv / avi / m4v / ogv / wmv，内置播放控件

浏览器内核无法解码的音视频编码会显示「此格式无法在内置播放器中播放」提示，此时可用标签栏的用默认应用打开改用系统播放器。

Office 文档：Word / PowerPoint / PDF ​

格式预览方式
.docx流式渲染的只读预览
.pptx幻灯片卡片化预览（只读），随面板宽度自适应缩放
.pdf由 WebView 内置阅读器渲染，具备翻页 / 缩放 / 搜索等阅读器自带能力
.doc / .ppt（旧版二进制格式）不支持内置预览，提示改用用默认应用打开

表格与数据：Excel / CSV / JSONL / Parquet / SQLite ​

Excel（xlsx / xls / xlsm）、CSV、JSONL/NDJSON、Parquet、SQLite（.db / .sqlite / .sqlite3）这几种格式统一走同一套表格预览：

  • 一个文件内有多个 sheet / 表时，底部标签栏可切换查看；Excel 单元格的加粗、字色、底色等样式也会保留
  • JSONL/NDJSON 按全文件的字段并集自动生成表头；个别解析失败的行会原文原样降级显示在单独一列，不影响其余行正常显示
  • 每个 sheet / 表最多渲染 1000 行，超出时顶部出现「仅显示前 N 行，共 M 行」提示
  • Parquet、SQLite 单文件超过 200MB 会拒绝加载并提示文件过大；SQLite 库中没有用户表时提示「此数据库没有用户表」

Excel 值级编辑 ​

仅 .xlsx 支持直接编辑单元格（xls / xlsm 仍为只读预览）：

  1. 双击目标单元格进入编辑，按 Enter 或失焦提交，Esc 取消
  2. 含公式的单元格不可编辑，双击会提示「公式单元格不可编辑」
  3. 已修改但未保存的单元格以琥珀色描边标记
  4. 按 Ctrl+S 或点击标签栏的保存按钮写回磁盘；首次保存前会自动生成一份 .bak 备份，误改可从备份恢复

SVG 与 Jupyter Notebook ​

  • SVG：预览态用 <img> 展示矢量图（不会执行文件内嵌的脚本），可切换到编辑态查看/修改源码
  • Jupyter Notebook（.ipynb）：只读渲染，Markdown / 代码单元格复用对话区同款高亮，单元格输出按类型分流为文本或图片；解析失败时给出内联提示，不影响切回源码编辑

字体 / 压缩包 / 十六进制 ​

格式预览方式
字体（ttf / otf / woff / woff2）中英文多字号样张；文件超过 50MB 拒绝加载
压缩包（zip / jar / tar / gz / tgz）零解压列出内部文件树；文件本身或 gz/tgz 解压后内容超过 200MB 时拒绝加载，条目数超过 5000 条时仅显示前 5000 条
压缩包（7z / rar / bz2）暂不支持内置预览，提示改用用默认应用打开
其余已知二进制格式（exe / dll / so / class 等）十六进制查看器（Hex + ASCII 双栏），仅读取文件前 256KB

预览失败时 ​

某个文件解析失败、格式暂不支持内置预览，或被判定为无法预览的二进制文件时，会显示对应的提示与出路（用默认应用打开 / 在文件夹中显示），而不是空白页；无法识别的二进制文件还会额外提供以十六进制查看的入口。

在对话中 @ 引用文件 ​

在对话输入框输入 @，弹出文件选择面板，选中文件后，文件内容以上下文的形式附加到本条消息，供 AI 参考。适合让 AI 分析特定文件时使用。

详细的 @ 引用用法见对话面板。
