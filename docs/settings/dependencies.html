依赖管理 ​

入口： 侧边栏 → 设置 → 依赖管理

依赖管理页面用于检测与安装应用所需的外部工具（CLI 及运行时）。相比 环境准备 中的首次安装引导，本页提供更完整的依赖状态查看与安装来源配置。

依赖分组 ​

依赖按重要性分组显示：

分组说明
AI CLI · 至少安装一个三个 AI CLI 家族 Claude / Codex / OpenCode 归为一组（v0.9.0 起），任务至少需要其中一个才能运行；三者全部未安装时，分组顶部显示醒目提示「请至少安装一个:Claude / Codex / OpenCode」
必须依赖其他核心工具，如 git
可选依赖扩展功能所需，缺少时部分功能受限

三个 AI CLI 都是可选其一

你不必装齐三个 CLI——按需选用即可。例如只用 Claude、或只用 OpenCode 配合第三方供应商，都能正常发起任务。只要三者中已安装任意一个，分组警告即消失。

依赖状态检测 ​

页面加载时自动扫描系统环境，每个依赖项显示以下信息：

  • 状态： 已安装的依赖显示绿色已安装标签；未安装则显示安装入口
  • 来源标签： 应用安装 / 系统安装 / 未安装
  • 版本号： 已安装时显示检测到的具体版本
  • 操作按钮（仅未安装时显示）： 支持应用内安装的依赖显示安装按钮（点击后变为安装中…）；不支持应用内安装的依赖显示触发系统安装按钮——点击后调起操作系统自带的安装流程（目前对应 macOS 上的 git：弹出系统的命令行开发者工具安装对话框），完成后回到本页重新检测即可

安装依赖 ​

点击依赖项右侧的安装按钮后，进度日志会实时展示在下方：

日志阶段含义
下载中 xx%正在从指定来源下载安装包
解压中…正在解压下载的文件
执行中…正在运行安装脚本
失败：安装失败，错误详情保留在日志区域

安装日志在关闭对话框后仍然保留，导航到其他页面再返回依然可以查看。

安装完成后需重启应用

新安装的工具需要重启应用才能被正确检测到，因为 PATH 环境变量在应用启动时已固定。请根据提示重启。

安装失败时，日志区域提供重试按钮。若反复失败，请参阅 故障排查 查看常见原因与解决方法。

安装来源配置 ​

入口： 依赖管理页面 → 安装来源 区域

安装来源决定应用从哪里下载 CLI 工具。提供三种模式：

来源说明适用场景
官方脚本使用各 CLI 官方提供的安装脚本网络直连、访问官方源无障碍时
GitHub 直连直接从 GitHub Releases 下载网络可访问 GitHub 时
GitHub 加速通过镜像服务转发 GitHub Releases国内网络访问 GitHub 受限时（推荐）

选择 GitHub 加速 时，还可在下方出现的输入框中填写自定义镜像前缀（v0.6.5 起支持），格式为 https://your-mirror.example.com/，留空则使用内置默认镜像。

国内网络建议

若你在中国大陆使用，GitHub 直连可能速度较慢或不稳定，建议选择 GitHub 加速 并使用内置默认镜像或自填可信镜像地址。

OpenCode CLI 的安装方式

OpenCode CLI（v0.9.0 起可在本页一键安装）在 Windows 上由应用自管从 GitHub 下载并解压安装（同样支持上述 GitHub 直连 / 加速来源）；在 macOS / Linux 上则调用官方安装脚本（opencode.ai/install）。安装完成后同样需按提示重启应用。

下载代理配置 ​

安装依赖时的下载请求支持独立于模型供应商的代理设置，同样是三态：

模式说明
跟随系统实时读取操作系统当前的代理设置（如 Clash 等代理工具接管系统代理后即可生效），默认模式
自定义单独为依赖下载指定代理地址
直连不使用任何代理，直接连接目标地址

TIP

下载代理与 模型供应商 → 网络代理 是两套独立的三态开关，互不影响——前者只管 CLI 安装包下载，后者管的是任务运行时 CLI 对该供应商发出的请求。
