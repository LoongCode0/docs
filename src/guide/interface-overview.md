# 界面导览

本页介绍 LoongCode 主界面的各个区域，帮你快速熟悉界面布局，找到你需要的功能。

## ① 左侧任务栏

左侧任务栏是管理工作区和任务的主要区域。从上到下依次包含：

- **折叠侧栏**：将鼠标悬浮到左上角 Logo 区域，可点击**收起侧栏**将任务栏折叠；点击**展开侧栏**恢复（v0.7.0 起支持）。
- **搜索框**：按关键词实时筛选任务。
- **＋ 打开工作区** / **＋ 新建任务**：搜索框下方的两个按钮，分别用于添加项目文件夹和快速新建一个任务草稿。
- **已置顶**：置顶的任务集合，独立显示于任务列表上方，不受「折叠全部」影响。
- **工作区树**：列出所有已打开的工作区（项目文件夹）。鼠标悬浮到工作区名称行，右侧出现**新建任务**、**查看文件**、**移除工作区**三个快捷图标；点击**查看文件**会将左侧栏切换为该工作区的**文件树**视图，点击文件树顶部的**返回任务**可退回任务列表。
- **任务列表**：每个工作区下方展示该工作区的任务条目。每行左侧的小圆点用**颜色**表示任务状态（空闲、运行中、成功、出错、提问中），并用**明暗**表示是否有未读消息——有新消息时圆点明亮，看过后变暗淡。详见[任务与工作区](/features/tasks-and-workspaces#任务状态)。
- **归档入口**：任务大组头右侧可切换到归档视图，查看或管理已归档的历史任务。

另外，左侧栏底部还有**新手引导**快捷入口、**定时总览**入口（时钟图标，集中管理所有任务的[定时执行](/features/scheduled-send)）和**打开设置**按钮。

在这里你可以切换任务、管理工作区，以及快速定位历史对话。详细说明见[任务与工作区](/features/tasks-and-workspaces)。

## ② 顶部标题栏

标题栏横跨界面顶部，包含：

- **当前工作区 / 分支切换器**：显示当前任务所在的工作区名称和 Git 分支（worktree 即 Git 的一种独立检出目录，允许同时在多个分支上工作）；点击分支名称可新建分支、切换分支或新建 worktree。
- **提交菜单**：位于顶部标题栏，提供**提交**与**更多 Git 操作**入口，可查看未提交的变更并完成提交。
- **更新徽标**：当检测到新版本时，右上角会出现**更新**徽标；点击后可立即重启完成更新。
- **后台活动指示灯（⚡N）**：当前任务有后台命令 / 子智能体在运行时，右侧出现 **⚡N** 指示灯（N = 正在运行的后台任务数）；点击可打开后台活动浮层。详见[后台任务活动](/features/background-tasks)。
- **侧边面板**按钮与**终端**按钮：分别切换右侧面板（详见下文 ⑤）和底部终端面板的显示状态。

在这里你可以快速切换 Git 分支、管理代码版本，以及检查应用更新。

## ③ 中央对话区

中央区域是与 AI 交互的主舞台，显示当前任务的完整对话历史：

- **消息气泡**：用户消息和 AI 回复以时间顺序排列；AI 回复实时流式渲染，Markdown 自动格式化。
- **思考过程块**：AI 的内部推理过程以折叠块形式呈现，标题为「思考过程」；点击可展开或收起。
- **工具卡片**：AI 调用工具（读写文件、执行命令等）时显示带状态的卡片；对代码的修改以 Diff（新旧代码对比）形式展示。
- **提问卡片**：需要用户授权时弹出，包含**允许** / **拒绝**按钮供你选择。
- **分屏**：可同时打开多个对话分栏，每栏独立运行一个任务（各栏有自己的输入框与 AI 会话）。

在这里你可以查看对话历史、审阅 AI 的操作，以及对 AI 的权限请求作出回应。

## ④ 输入框

输入框位于界面底部，是向 AI 发送消息的入口：

- **选择器条**（输入框上方）：从左到右依次为 CLI 选择（Claude CLI / Codex CLI / OpenCode CLI）、模型选择、思考深度（Claude：low / medium / high / xhigh / max / ultracode；Codex：minimal / low / medium / high / xhigh；OpenCode：default / minimal / low / medium / high / xhigh）、执行授权（Claude：default / plan / acceptEdits / auto / dontAsk / bypassPermissions；Codex：只读 / 自动 / 完全访问；OpenCode：auto / ask）、当前分支。
- **上下文用量指示器**：选择器条最右侧显示已用 / 总量的上下文 token 比例（token 是 AI 处理文本的计量单位）；可点击**压缩**对话历史以释放空间。
- **触发器**：在输入框中输入 `@` 可从工作区文件列表添加文件上下文，输入 `/` 可打开命令面板使用内置命令、插件命令和技能（技能和命令统一从 `/` 调用）。
- **占位提示**：「向 Claude 提问，输入 @ 添加文件，/ 使用命令，$ 使用技能」；其中 `$ 使用技能` 只是对技能类命令的说明文字，技能实际同样通过 `/` 命令面板调用（详见[斜杠命令](/features/slash-commands)）。等待 AI 回答时占位提示变为「请在上方卡片作答;或在此草拟下一条消息…」。

在这里你可以配置每次对话的 AI 参数，以及向 AI 发送指令。详细说明见[对话面板](/features/conversation)。

## ⑤ 右侧面板

右侧面板（点击标题栏**侧边面板**按钮切换显示）是文件查看与代码审阅的主区域。标签栏由两类标签组成：

| 标签类型 | 说明 |
|----------|------|
| **文件标签**（动态） | 每个打开的文件对应一个标签，使用 Monaco 编辑器展示，支持语法高亮、Diff 对比与 Markdown 预览；标签标题为文件名，有未保存修改时显示脏点 |
| **审查** | 查看工作区的代码 Diff（未暂存 / 已暂存 / 全部分支更改 / 上一轮更改），确认后可提交 |
| **浏览器** | 内嵌浏览器，可在应用内打开网页，并把网页元素直接加入对话上下文 |

打开文件的方式：在左侧栏的文件树中**单击文件节点**，文件即在右侧面板新建文件标签打开；也可按 **Ctrl+P** 搜索并打开文件，或点击标签栏右侧的 **+** 按钮选择「打开文件」。

**文件树**不是右侧面板的标签页，而是**左侧栏**的一种视图：在工作区行点击**查看文件**即可进入，点击顶部**返回任务**可退回任务列表。

**终端**同样不属于右侧面板：它是从中央对话区底部弹出的独立面板，点击标题栏的**终端**按钮（快捷键 Ctrl+`）即可打开，直接在工作区目录运行命令。

::: tip 浏览器标签
在浏览器标签中点击 **+** 可新增一个内嵌浏览器窗口；点击**选中元素加入聊天**，可将网页上的元素直接发送给 AI 进行分析。
:::

在这里你可以查看和编辑项目文件、审阅 AI 的代码改动，以及配合内嵌浏览器做前端调试。详细说明见[文件与编辑器](/features/files-and-editor)。

## ⑥ 设置入口

设置入口位于左侧任务栏底部，图标为齿轮形状，点击即可打开设置页面。

设置页面提供以下分类（左侧导航）：

- **常规**：主题、数据目录、CLI 路径、网络代理、提示音、代码快照、自动压缩、使用引导等
- **依赖管理**：检测并安装 Claude CLI / Codex CLI / OpenCode CLI 等依赖
- **模型供应商**：配置各 AI 服务商的 API 密钥（API 密钥是服务商发给你的专属访问凭证）、接口地址、可用模型列表
- **插件**：发现、安装和管理插件
- **技能**：创建或编辑可复用的提示词技能
- **MCP 服务器**：增删改 MCP（Model Context Protocol）服务器配置，扩展 AI 可用的外部工具
- **子智能体**：管理子智能体配置
- **使用统计**：查看 token 用量、费用、活跃热力图等

详细说明见[设置 → 常规](/settings/general)。
