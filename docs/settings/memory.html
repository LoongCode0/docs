记忆 ​

入口： 侧边栏 → 设置 → 记忆

记忆（Memory）页面集中管理 Claude / Codex / OpenCode 用于长期记住项目约定与个人偏好的持久化指令文件（CLAUDE.md / AGENTS.md），以及 Claude 的 auto-memory（模型在对话过程中自动积累的分主题笔记）。

本页通过顶部 Claude | Codex | OpenCode 切换栏分别维护各 CLI 的记忆文件。

全局记忆与项目记忆 ​

页面按作用域分两组展示：

作用域说明对应文件
全局记忆跨项目生效，存储在各 CLI 的用户级配置目录Claude：~/.claude/CLAUDE.md；Codex：~/.codex/AGENTS.md；OpenCode：~/.config/opencode/AGENTS.md（遵循 XDG_CONFIG_HOME 环境变量，未设置时使用该默认路径）
项目记忆仅当前工作区生效；未打开工作区时该组显示提示「选择工作区任务后可查看项目记忆」Claude：<工作区>/CLAUDE.md；Codex / OpenCode：<工作区>/AGENTS.md

若对应文件已存在，显示一行记录（文件名、路径、文件大小），点击可打开详情弹窗；若尚未创建，显示「尚未创建」提示与新建按钮，点击即在目标路径创建一份空文件。

Codex 的 override 文件优先

Codex 侧（全局或项目）若存在同目录下的 AGENTS.override.md，Codex CLI 实际读取的是这份 override 文件而非 AGENTS.md——本页会据此优先展示并编辑它。

AUTO-MEMORY（仅 Claude · 项目记忆） ​

切到 Claude 且已打开工作区时，「项目记忆」组下方额外展示一个 AUTO-MEMORY 子列表——这是 Claude 在对话过程中自动写入的分主题记忆文件，路径为 ~/.claude/projects/<项目编码路径>/memory/，与 Claude Code 自己读写的目录完全一致，因此这里看到的就是模型实际会加载的记忆。

  • 每条记录展示一个主题笔记的文件名与大小；其中索引文件 MEMORY.md 额外带索引徽标
  • 页面顶部的搜索框按文件名或正文内容过滤，命中范围覆盖全局记忆、项目记忆与全部 auto-memory 笔记
  • 点击新建记忆手动新建一个主题笔记：输入文件名（无需手动加 .md 后缀，应用会自动补齐，且会拒绝包含路径分隔符等非法字符的名称）后创建

查看与编辑记忆文件 ​

点击任意一条记忆记录，弹出详情弹窗：

  • 默认预览模式，以渲染后的 Markdown 展示内容（auto-memory 文件开头的 YAML frontmatter 会包成代码块展示，避免被 Markdown 误解析成加粗标题）
  • 点击弹窗顶部的铅笔图标切换到编辑模式（Monaco 代码编辑器），修改后点击保存按钮或按 Ctrl+S 写回磁盘
  • 在文件管理器中显示图标可定位到系统文件管理器中的原文件
  • 仅 auto-memory 笔记提供删除（垃圾桶图标）入口，点击后需二次确认，确认文案为「确定删除「{名称}」？文件将移入回收站。」；全局 / 项目主记忆文件（CLAUDE.md / AGENTS.md）不提供删除入口
  • 有未保存改动时关闭弹窗会弹出「放弃未保存的修改？」确认，避免误丢改动
