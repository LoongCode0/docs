模型供应商 ​

入口： 侧边栏 → 设置 → 模型供应商

模型供应商功能让你通过第三方 API 或自建代理接入模型服务，作为官方账号之外的另一条接入路径。配置供应商后，可在任务选择器栏中选择对应的模型档位，调用该供应商提供的模型。

什么时候需要配置供应商 ​

如果你已通过 claude login 等方式完成官方账号登录，默认即可正常使用，无需配置本页任何内容。 以下场景才需要配置供应商：

  • 降低费用 — 通过 deepseek、z.ai 等第三方平台调用模型，价格通常低于官方 API
  • 无法访问官方 API — 所在网络环境连不上官方服务时，改走可达的第三方接口
  • 私有代理 / 自建网关 — 企业内部统一管理模型流量，CLI 请求需经过自建的兼容代理

供应商类型 ​

应用将供应商分为三类：

类型说明
原生（OAuth）Claude OAuth 与 Codex OAuth，凭据由 CLI 自身登录管理，应用仅管理模型清单与上下文窗口配置
内置预置的第三方平台，开箱即用，填入 API Key 即可
自定义手动填写接口地址与 API Key，适用于任意 OpenAI/Anthropic 兼容代理

内置供应商 ​

应用预置了以下 8 家第三方平台，无需手动填写接口地址，直接输入 API Key 即可启用：

供应商名称说明
deepseekDeepSeek 平台，支持 Anthropic 与 OpenAI 双接口
z.ai智谱 z.ai 平台，仅 Anthropic 接口
bigmodel智谱 BigModel 平台，支持 Anthropic 与 OpenAI 双接口
openrouterOpenRouter 聚合平台，支持 Anthropic 与 OpenAI 双接口
moonshot月之暗面 Kimi 平台，支持 Anthropic 与 OpenAI 双接口
minimaxMiniMax 平台，支持 Anthropic 与 OpenAI 双接口
mimo小米 Mimo 平台，仅 Anthropic 接口
Qwen阿里云通义千问平台，支持 Anthropic 与 OpenAI 双接口

供应商可用性 ​

每个供应商面板右侧会显示可用性指示圆点（v0.9.0 起为三项）：

  • Claude CLI 可用 — 当 Anthropic 接口地址非空时显示
  • Codex CLI 可用 — 当 OpenAI 接口地址非空时显示
  • OpenCode CLI 可用 — 当 OpenAI 接口地址非空时显示（OpenCode 走 OpenAI 兼容端点，可用条件与 Codex 相同）

仅支持单接口的供应商（如 z.ai、mimo）只对相应 CLI 可用；只填了 OpenAI 接口地址的供应商，可同时供 Codex 与 OpenCode 使用。

配置供应商 ​

OAuth 供应商无需配置 API Key

Claude OAuth 与 Codex OAuth 的凭据由 CLI 自身的登录流程管理（如 claude login），在本页无需也无法填写 API Key。以下步骤适用于内置与自定义供应商。

1. 选择供应商 ​

在左侧供应商列表中点击要配置的供应商名称，右侧显示详情面板。

2. 填写 API Key ​

在 API Key 输入框中粘贴从对应平台获取的密钥。

请妥善保管 API Key

API Key 是访问模型服务的唯一凭据，泄露后可能导致费用损失或数据风险。不要将 API Key 截图分享、写入版本库，或通过不安全渠道传输。

3. 配置接口地址（可选） ​

内置供应商已预填接口地址，通常无需修改。若平台更换域名或你使用私有部署版本，可在以下字段中修改：

字段说明
Anthropic 接口地址用于 Claude CLI；支持 Anthropic 消息格式的接口根地址
OpenAI 接口地址用于 Codex CLI；支持 OpenAI Chat Completion 格式的接口根地址

TIP

两个接口地址不能同时为空，否则保存时会提示错误。

4. 配置模型映射 ​

各家平台的模型命名千差万别（如 deepseek-v4-pro、glm-5、kimi-k2.6），LoongCode 用统一的档位概念屏蔽这种差异：你在任务中只需选择 Opus / Sonnet / Haiku 档位，实际调用的则是当前供应商在该档位槽上绑定的具体模型。

模型映射就是把这三个档位槽与供应商实际提供的模型名称对应起来。

档位对应角色
Opus复杂任务
Sonnet常规任务
Haiku轻量任务

在每个档位的下拉列表中选择想要绑定的具体模型。若某档位留空（显示"未设置"），则该档位在此供应商下不可用。

5. 配置上下文窗口（可选） ​

上下文窗口字段允许手动指定模型支持的 Token 数量上限，用于上下文使用率显示与自动压缩触发判断。留空则使用应用内置的默认值（v0.5.2 起出厂预设了主流模型的默认值；未匹配到的模型回退到 200,000 Token）。

添加自定义供应商 ​

若你的目标平台不在内置列表中（例如自建 OpenAI 兼容代理），可添加自定义供应商：

  1. 点击左侧供应商列表底部的"添加"按钮，列表中出现名为新供应商的条目。
  2. 点击名称旁的编辑（铅笔）图标，输入供应商的显示名称。
  3. 按 配置供应商 步骤填写接口地址、API Key 与模型映射。

自定义供应商可通过列表底部的删除按钮移除（需二次确认）；内置供应商与 OAuth 供应商不可删除。

Codex 内置中转 ​

若已安装 Codex CLI，部分非 OAuth 原生供应商会出现启用 Codex 中转开关。开启后，Codex CLI 发出的 Responses API 请求将经由应用内置的中转层翻译为 Chat Completions 格式转发至该供应商，响应再反向翻译回 Responses 格式返回给 Codex，从而让 DeepSeek 等只提供 Chat Completions 接口的服务也能驱动 Codex 任务（v0.7.1 起）。

OpenCode 供应商复用 ​

OpenCode 任务的模型下拉直接复用你在本页配置的 OpenAI 兼容供应商（v0.9.0 起）：应用会把供应商的接口地址与模型映射经由托管配置注入 OpenCode，无需在 OpenCode 侧重复填写 API Key。因此 deepseek、GLM 等第三方模型可直接用于 OpenCode 任务。

  • 只要某个供应商填好了 OpenAI 接口地址（可用性圆点显示「OpenCode CLI 可用」），它的模型就会出现在 OpenCode 任务的模型下拉中。
  • OpenCode 没有原生 OAuth 档位，模型下拉只列出供应商模型；只填了 Anthropic 接口地址、没填 OpenAI 接口地址的供应商，其模型在 OpenCode 的下拉中会置灰（原因提示「供应商未配置接口地址」）。
