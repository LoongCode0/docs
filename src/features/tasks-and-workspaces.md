# 任务与工作区

LoongCode 用**工作区**（Workspace，对应你磁盘上的项目目录）和**任务**（Task，对应一次 AI 对话会话）两级结构来组织你的工作。本页介绍如何添加工作区、管理任务生命周期，以及置顶、搜索、归档等高效操作。

> 如果你还没开始第一个任务，建议先看[快速上手](/guide/quick-start)了解基础操作流程。

## 左侧任务栏布局

左侧任务栏从上到下的真实元素顺序为：

| 区域 | 说明 |
|------|------|
| **Logo / 折叠按钮** | 悬浮后可折叠侧栏（v0.7.0 起） |
| **搜索框** | 按关键词实时筛选任务 |
| **＋ 打开工作区** | 添加项目文件夹为工作区 |
| **＋ 新建任务** | 快速新建一个任务草稿 |
| **已置顶** | 已置顶任务集合，独立于分组折叠 |
| **任务大组头** | 含「任务 / 归档」标签与视图切换（分组列表 / 时间线） |
| **工作区树 + 任务列表** | 按工作区分组，展示各任务 |
| **底部栏** | 新手引导、移动端支持、打开设置 |

::: tip 快捷键
- `Ctrl+N`：新建任务草稿（任意界面可用）
- 悬浮 Logo 点击：折叠 / 展开左侧任务栏（v0.7.0）
:::

## 添加与管理工作区

### 添加工作区

1. 点击搜索框下方的**＋ 打开工作区**按钮。
2. 在弹出的文件夹选取对话框中选择本地项目目录，点击**确认**。
3. 如果该目录下有 Claude Code 历史会话，会弹出询问「**同步会话**」——点**导入**可将历史记录导入为任务；点**跳过**则跳过。

工作区添加成功后会出现在任务列表树中。

### 工作区操作

鼠标悬浮到工作区名称行，右侧出现三个快捷图标：

- **新建任务**：在该工作区下新建任务
- **查看文件**：将左侧栏切换为该工作区的**文件树**视图（文件树顶部的**返回任务**按钮可退回任务列表）
- **移除工作区**：从 LoongCode 中移除该工作区条目（**不影响磁盘上的实际文件**）

::: warning 移除工作区
移除工作区会删除该工作区下的所有任务记录，但磁盘上的代码与历史文件不受影响。
:::

## 新建任务

### 草稿态机制

点击**＋ 新建任务**（或按 `Ctrl+N`）后，任务进入**草稿态**——此时尚未写入数据库，主区域内联显示配置选择器供你挑选工作区、模型、Git 分支或 worktree（Git worktree 是 Git 的独立检出目录，允许同时在多个分支上工作）。

**发送第一条消息后，任务才正式创建**，这样可以避免空任务堆积。

草稿会默认选中上次用过的工作区，少点一步。草稿选中的工作区被外部删除时，发送按钮会被禁用并给出提示。

### 草稿态下可用功能

即使任务还在草稿态，左侧栏的文件树，右侧面板的文件标签、审查、浏览器，以及终端均可正常使用，它们按工作区工作，不依赖任务是否已落盘。

## 任务状态

每个任务行左侧都有一个小圆点，它同时传递两层信息：用**颜色和光晕**表示任务所处的状态，用**明暗**表示这个任务有没有你还没看过的新消息。

### 状态颜色

圆点的颜色对应任务当前状态，共五种：

| 指示灯 | 状态 | 圆点外观 | 含义 |
|:---:|------|---------|------|
| <span class="lc-dot lc-dot--idle"></span> | **空闲** | 绿色，无光晕 | 没有正在运行的回合，等待你的下一条消息 |
| <span class="lc-dot lc-dot--running"></span> | **运行中** | 琥珀色（橙黄），带柔和光晕 | AI 正在生成回复或执行工具 |
| <span class="lc-dot lc-dot--success"></span> | **成功** | 绿色，无光晕 | 最近一轮回合正常结束 |
| <span class="lc-dot lc-dot--error"></span> | **出错** | 红色，无光晕 | 最近一轮回合以错误结束 |
| <span class="lc-dot lc-dot--asking"></span> | **提问中** | 蓝色，带柔和光晕 | AI 正在等你回答一个问题，需要你处理 |

> 上表「指示灯」列即为各状态圆点在浅色界面下的实际外观（深色模式下颜色更亮）。

::: tip 关于圆点外观的几个细节
- 圆点始终是**静态实色**，不会闪烁、旋转或跳动；「运行中」和「提问中」是通过**向外发散的柔和光晕**来吸引你的注意，而不是靠动画。
- 「空闲」和「成功」都显示为绿色、都没有光晕，外观一致——「成功」只表示刚结束的那一轮没有报错，之后任务随即回到等待输入的「空闲」状态。
- 「提问中」的优先级最高：只要 AI 抛出了等待你回答的问题，圆点就立刻变为蓝色并发光，无论任务底层处于哪种状态。
:::

### 圆点的明暗：未读提示

除了颜色，圆点的**明暗**用来提示这个任务有没有你还没查看的新消息：

| 指示灯 | 明暗 | 含义 |
|:---:|------|------|
| <span class="lc-dot lc-dot--idle"></span> | 明亮（满色） | 该任务有未读的新消息（例如 AI 在后台完成了回复，而你正在查看别的任务） |
| <span class="lc-dot lc-dot--idle lc-dot--read"></span> | 暗淡（半透明） | 你已经查看过该任务的最新消息，圆点随之变暗，表示「已读」 |

切换到某个任务、看到它的最新回复后，它的圆点就会从明亮转为暗淡。

::: tip 「运行中」始终明亮
琥珀色的「运行中」圆点是一个例外——只要任务还在运行，它就始终保持明亮、不会因「已读」而变暗，方便你随时一眼看出哪些任务正在后台工作。
:::

## 任务排序与置顶

### 按最近活动排序

任务列表与时间线分桶按**最近活动时间**排序——刚发过消息或刚结束一轮对话的任务会自动浮到前面（v0.4.2 起）。

### 置顶任务

置顶通过任务行上的**快捷按钮**完成（不在右键菜单里）：

1. 将鼠标悬浮到任务行上——行首的状态圆点会变为一个图钉形的**置顶任务**按钮。
2. 点击该按钮即完成置顶。

置顶后任务出现在**已置顶**分组，跨视图、不受「折叠全部」影响。已置顶的任务悬浮时该按钮变为**取消置顶**，再点一次即可恢复。

::: tip
置顶任务受自动归档保护，不会被自动归档。
:::

## 搜索任务

搜索框（占位提示「搜索任务…」）位于侧栏顶部 Logo 区域下方，输入关键词后任务列表实时筛选。

- 无匹配时显示「未找到匹配任务」
- 点击搜索框右侧的**清除搜索**按钮可清空关键词

## 归档任务

### 手动归档

右键点击任务行，选择**归档**即可（任务行悬浮时行尾的**任务操作**下拉菜单中同样可以归档）。

归档后任务从主列表移出，进入归档视图，主列表更加清爽。

取消归档：进入归档视图后，右键任务选**取消归档**，任务的活动时间会被重置，回到主列表最上方，方便立即继续。

### 自动归档

应用**启动时**与**每小时**各跑一次自动归档，按「最近活动时间」超过阈值自动将久未触碰的任务归档（v0.4.3 起）。

**保护规则**：运行中任务和已置顶任务不会被自动归档。

**调整阈值**：在**设置 → 常规**中可开关自动归档，并设置阈值数值与单位（小时 / 天 / 月），默认开启、7 天。

### 查看归档视图

1. 在任务大组头右侧点击**查看归档**。
2. 进入归档视图后，任务按归档时间降序排列。
3. 归档视图下自动隐藏置顶区和新建入口，没有归档任务时显示「暂无归档任务」。
4. 点击**退出归档**返回主任务列表。

## 任务右键菜单

右键任务行可弹出操作菜单，仅包含以下三类操作：

| 菜单项 | 说明 |
|------|------|
| **重命名** | 仅未归档任务显示 |
| **归档** / **取消归档** | 按任务当前归档状态二选一 |
| **删除** | 危险操作，需二次确认 |

置顶不在右键菜单中——它是任务行悬浮时行首出现的图钉快捷按钮，见上文「置顶任务」。

::: tip
任务行悬浮时行尾还会出现**任务操作**下拉菜单，它提供与右键菜单相同的操作。
:::

::: danger 删除任务
删除操作不可恢复，请确认后再继续。删除任务仅删除 LoongCode 内的对话记录，不影响磁盘上的项目代码。
:::

## 视图模式

任务大组头提供两种视图切换：

- **分组列表**：按工作区分组展示，可展开 / 折叠各工作区
- **时间线**：按时间桶（今天、昨天、本周……）排列，跨工作区浏览

::: tip 选中任务自动定位（v0.8.0 起）
无论当前处于哪种视图，当你（在分屏中切换分栏、或通过机器人 / 移动端等方式）选中一个任务时，左侧栏会**自动展开并滚动定位到它所在的分组**——已置顶任务定位到「已置顶」组，其余按当前视图定位到对应的工作区分组或时间线日期桶，省去手动翻找折叠分组的麻烦。
:::
