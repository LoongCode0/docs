模型供应商 ​

入口： 侧边栏 → 设置 → 模型供应商

模型供应商功能让你通过第三方 API 或自建代理接入模型服务，作为官方账号之外的另一条接入路径。配置供应商后，可在任务选择器栏中选择对应的模型档位，调用该供应商提供的模型。

什么时候需要配置供应商 ​

如果你已通过 claude login 等方式完成官方账号登录，默认即可正常使用，无需配置本页任何内容。 以下场景才需要配置供应商：

  • 降低费用 — 通过 deepseek、z.ai 等第三方平台调用模型，价格通常低于官方 API
  • 无法访问官方 API — 所在网络环境连不上官方服务时，改走可达的第三方接口
  • 私有代理 / 自建网关 — 企业内部统一管理模型流量，CLI 请求需经过自建的兼容代理

官方账号登录也可以直接在本页完成

不必再切到终端敲 claude login——本页的 Claude OAuth / Codex OAuth 卡片自带登录区块，点一下即可在应用内完成登录，操作的是同一份本机凭据。

供应商类型 ​

应用将供应商分为三类：

类型说明
原生（OAuth）Claude OAuth 与 Codex OAuth，凭据由 CLI 自身登录管理，应用仅管理模型清单与上下文窗口配置
内置预置的第三方平台，开箱即用，填入 API Key 即可
自定义手动填写接口地址与 API Key，适用于任意 OpenAI/Anthropic 兼容代理

内置供应商 ​

应用预置了以下 8 家第三方平台，无需手动填写接口地址，直接输入 API Key 即可启用：

供应商名称说明

z.ai智谱 z.ai 平台，仅 Anthropic 接口
bigmodel智谱 BigModel 平台，支持 Anthropic 与 OpenAI 双接口

供应商可用性 ​

每个供应商面板右侧会显示可用性指示圆点（v0.9.0 起为三项）：

  • Claude CLI 可用 — 当 Anthropic 接口地址非空时显示
  • Codex CLI 可用 — 当 OpenAI 接口地址非空时显示
  • OpenCode CLI 可用 — 当 OpenAI 接口地址非空时显示（OpenCode 走 OpenAI 兼容端点，可用条件与 Codex 相同）

仅支持单接口的供应商（如 z.ai、mimo）只对相应 CLI 可用；只填了 OpenAI 接口地址的供应商，可同时供 Codex 与 OpenCode 使用。

OAuth 供应商登录 ​

Claude OAuth 与 Codex OAuth 卡片不填 API Key，而是提供一个专属的登录区块：

  • 状态灯：绿色表示已登录（悬浮或文字处显示登录账号信息）、灰色表示未登录、黄色闪烁表示正在查询状态或正在登录
  • 登录 / 重新登录：点击后应用在该供应商已配置的代理环境下后台执行对应 CLI 的登录子命令，CLI 会自行弹出浏览器完成账号授权；登录过程中可点击取消中止
  • 登出：仅已登录时可点击，登出当前账号

登录 / 登出 / 状态三条命令均可编辑：这里编辑的只是子命令（默认对应官方 CLI 的 login / logout / 状态查询子命令），实际调用的可执行文件取自 常规设置 → CLI 命令。第三方发行版（如 reclaude）的子命令可能与官方不同，按需修改，失焦即自动保存；登录 / 登出 / 状态三项均不能留空，否则会提示错误。

Claude OAuth 卡片还有第 4 条「用量」命令

仅 Claude OAuth 卡片额外提供一条可编辑的用量命令（默认对应 CLI 的用量查询子命令），供下文账号用量读取真实数据；Codex OAuth 没有这个输入框——它的用量走应用内置的逆向接口，无需自定义命令。

账号用量 ​

已登录 OAuth 账号后，可以在应用内直接查看该账号当前的实时用量（即官方账号的速率限额消耗情况），无需切到终端手动执行用量命令。

与「使用统计」是两回事

本节的账号用量是实时查询账号侧的速率限额（对应官方账号页 / CLI 用量命令的数据，随官方计费周期滚动重置）；设置里的使用统计则是本地历史文件的离线 Token 统计，两者数据来源、口径都不同，互不影响。

两处可以看到 ​

  • 该供应商详情页内联展示：已登录的 Claude OAuth / Codex OAuth 卡片，登录区块下方会展示一组用量行（见下文数据说明）。
  • 侧边栏账号用量徽标：左侧任务栏底部（主题切换按钮左边）常驻一枚环形徽标，取 Claude / Codex 两个账号中较紧张的用量百分比决定圆环颜色与数字；两个账号都未登录或未开启用量时圆环显示空态（仅底环 + —）。悬浮徽标展开浮层，分别显示两个账号各自的完整用量（与详情页复用同一份数据，5 分钟轮询刷新，悬浮时立即补一次刷新）。

可以整体关闭（v0.9.9 起）

若不需要用量展示，或不希望应用每 5 分钟在后台拉起一次 CLI 查询，可在常规设置 → 账号用量中关闭总开关。关闭后侧边栏徽标整体隐藏、后台轮询与手动刷新全部停止，本页登录卡片的用量区块也会显示「账号用量已在「通用」设置中关闭」的提示。

数据说明 ​

用量以多行「窗口」形式展示，每行包含标签、百分比、进度条与重置倒计时：

行说明
会话（N 小时）当前会话窗口的用量占比
全部模型 / 单独限额（N 天 / N 个月）按模型分类的周期性限额占比；官方账号通常还会对个别模型单列一条「{模型名} 单独限额」

窗口时长文案按接口返回的实际滚动周期动态换算（如 5 小时 / 7 天 / 1 个月），不是写死的固定档位；用量占比达到较高水位时，百分比与进度条颜色会变为警示色。

Codex 用量为实验性功能 ​

Codex 官方未提供公开的用量查询接口，这里的数据来自逆向接口。因此 Codex OAuth 卡片默认不开启用量显示，登录后会看到一行提示「用量：未开启（实验性 · 基于逆向接口）」与开启按钮；点击开启后才会开始查询并展示 Codex 用量，同时常驻一行风险提示「数据来自逆向接口，非官方支持，可能随时失效」。Claude OAuth 的用量走官方支持的正式命令，无需该开关。

配置供应商 ​

OAuth 供应商无需配置 API Key

Claude OAuth 与 Codex OAuth 的凭据由上文登录区块或 CLI 自身的登录流程管理，在本页无需也无法填写 API Key。以下步骤适用于内置与自定义供应商。

1. 选择供应商 ​

在左侧供应商列表中点击要配置的供应商名称，右侧显示详情面板。

2. 填写 API Key ​

在 API Key 输入框中粘贴从对应平台获取的密钥。

请妥善保管 API Key

API Key 是访问模型服务的唯一凭据，泄露后可能导致费用损失或数据风险。不要将 API Key 截图分享、写入版本库，或通过不安全渠道传输。

3. 配置接口地址（可选） ​

内置供应商已预填接口地址，通常无需修改。若平台更换域名或你使用私有部署版本，可在以下字段中修改：

字段说明
Anthropic 接口地址用于 Claude CLI；支持 Anthropic 消息格式的接口根地址
OpenAI 接口地址用于 Codex CLI；支持 OpenAI Chat Completion 格式的接口根地址

TIP

两个接口地址不能同时为空，否则保存时会提示错误。

4. 配置模型映射 ​

各家平台的模型命名千差万别（如 deepseek-v4-pro、glm-5、kimi-k2.6），LoongCode 用统一的档位概念屏蔽这种差异：你在任务中只需选择 Opus / Sonnet / Haiku 档位，实际调用的则是当前供应商在该档位槽上绑定的具体模型。

模型映射就是把这三个档位槽与供应商实际提供的模型名称对应起来。

档位对应角色
Opus复杂任务
Sonnet常规任务
Haiku轻量任务

在每个档位的下拉列表中选择想要绑定的具体模型。若某档位留空（显示"未设置"），则该档位在此供应商下不可用。

5. 配置上下文窗口（可选） ​

上下文窗口字段允许手动指定模型支持的 Token 数量上限，用于上下文使用率显示与自动压缩触发判断。留空则使用应用内置的默认值（v0.5.2 起出厂预设了主流模型的默认值；未匹配到的模型回退到 200,000 Token）。

网络代理 ​

每个供应商详情面板底部都有一组独立的网络代理三态开关（Claude OAuth / Codex OAuth 同样具备），决定该供应商的 Claude CLI / Codex CLI 调用（含登录）走哪种代理，与其他供应商互不影响：

模式说明
跟随系统实时读取操作系统当前的代理设置（如 Clash 等工具接管系统代理），默认模式
自定义为该供应商单独指定代理地址与忽略代理的地址
直连不使用任何代理

选择自定义后展开两个字段（失焦即保存）：

字段说明
代理地址必须以 http:// 或 https:// 开头，否则保存时提示格式错误
忽略代理的地址逗号分隔的主机 / 域名列表，这些地址不经过代理；localhost 已默认放行；未填地址时按跟随系统处理

代理改动在下一条消息生效，无需重启应用或重新连接任务。

与「依赖管理」的下载代理是两套独立开关

本节代理只影响任务运行时 CLI 对该供应商发出的请求；应用自身下载 CLI 安装包走另一套配置，见 依赖管理 → 下载代理配置。

暂不覆盖 OpenCode CLI

本节代理三态目前只作用于 Claude CLI 与 Codex CLI（含二者的 OAuth 登录）；OpenCode 任务走的是系统环境自身的网络设置，暂不经过这里的代理配置。

添加自定义供应商 ​

若你的目标平台不在内置列表中（例如自建 OpenAI 兼容代理），可添加自定义供应商：

  1. 点击左侧供应商列表底部的"添加"按钮，列表中出现名为新供应商的条目。
  2. 点击名称旁的编辑（铅笔）图标，输入供应商的显示名称。
  3. 按 配置供应商 步骤填写接口地址、API Key 与模型映射。

自定义供应商可通过列表底部的删除按钮移除（需二次确认）；内置供应商与 OAuth 供应商不可删除。

Codex 内置中转 ​

若已安装 Codex CLI，部分非 OAuth 原生供应商会出现启用 Codex 中转开关。开启后，Codex CLI 发出的 Responses API 请求将经由应用内置的中转层翻译为 Chat Completions 格式转发至该供应商，响应再反向翻译回 Responses 格式返回给 Codex，从而让 DeepSeek 等只提供 Chat Completions 接口的服务也能驱动 Codex 任务（v0.7.1 起）。

模型目录同步 ​

中转层能否正确识别第三方模型 ID，依赖应用本地保存的一份 Codex 模型目录。Codex OAuth 供应商卡片提供更新模型ID列表按钮：点击后调用 CLI 拉取最新目录并与本地对比，完成后提示新增 / 移除的数量（已是最新则提示「模型列表已是最新」）。

若目录一直未同步过，供应商列表中 Codex OAuth 条目右侧会出现一个提示图标，悬浮显示「模型目录尚未同步，中转到第三方模型可能失败」，点击更新模型ID列表按钮同步即可消除。

OpenCode 供应商复用 ​

OpenCode 任务的模型下拉直接复用你在本页配置的 OpenAI 兼容供应商（v0.9.0 起）：应用会把供应商的接口地址与模型映射经由托管配置注入 OpenCode，无需在 OpenCode 侧重复填写 API Key。因此 deepseek、GLM 等第三方模型可直接用于 OpenCode 任务。

  • 只要某个供应商填好了 OpenAI 接口地址（可用性圆点显示「OpenCode CLI 可用」），它的模型就会出现在 OpenCode 任务的模型下拉中。
  • OpenCode 没有原生 OAuth 档位，模型下拉只列出供应商模型；只填了 Anthropic 接口地址、没填 OpenAI 接口地址的供应商，其模型在 OpenCode 的下拉中会置灰（原因提示「供应商未配置接口地址」）。
