插件 ​

入口： 侧边栏 → 设置 → 插件

插件（Plugin）是由社区或第三方开发者发布的功能扩展包，安装后可为 Claude CLI 或 Codex CLI 增加新的斜杠命令和工作流。安装的插件提供的命令会出现在 / 命令面板中，附带 plugin 来源标识。

本页通过 Claude | Codex 切换栏分别管理两个 CLI 的插件。页面内容分为三个标签页：发现、已安装、市场。

发现 ​

发现 标签页列出当前已添加的市场中所有可安装的插件。

每张插件卡片显示名称、描述及来源市场标签。右侧提供安装操作：

  • Claude 模式：点击 安装 按钮后弹出作用域选择菜单，可选 用户、项目 或 本地（后两项需先关联工作区任务）
  • Codex 模式：点击 安装 按钮直接以用户作用域安装，无作用域菜单

作用域说明：

作用域生效范围
用户当前用户的所有工作区
项目当前工作区所在项目（需关联工作区任务）
本地仅限本地，不随项目同步（需关联工作区任务）

若搜索栏输入关键词，结果会实时过滤插件名称和描述。

已安装 ​

已安装 标签页列出已安装的全部插件，显示名称、版本号及作用域标签。

每行右侧提供两个操作：

  • 启用开关 — 即时启用或禁用插件；禁用后该插件提供的命令在 / 面板中不再显示
  • 删除按钮（垃圾桶图标）— 删除插件并弹出确认对话框，确认后不可撤销

更新插件需先删除再重装

已安装列表中没有单独的"升级"按钮。若需更新插件版本，请先删除旧版本，再从 发现 标签页重新安装最新版本。删除操作通过对应 CLI 自身的插件管理命令完成，具体清理范围由 CLI 决定；如果你对该插件做过自定义配置，删除前请自行备份。

插件的命令在 / 命令面板中带有 plugin 来源标识，详见 斜杠命令。

市场 ​

市场 标签页管理插件来源，即从哪些仓库拉取可安装的插件列表。

添加市场 ​

在顶部输入框填写市场来源后点击 添加市场：

  • Claude 模式 示例：anthropics/claude-plugins-official、https://github.com/org/repo.git、/path/to/local-marketplace
  • Codex 模式 示例：openai/plugins

市场列表 ​

每条市场记录显示名称、可用插件数量及最近更新时间。

每行右侧提供以下操作（按条件显示）：

  • 刷新快照按钮（刷新图标）— 仅 Codex 模式下的 Git 类型市场显示，用于拉取该市场仓库的最新提交
  • 删除按钮（垃圾桶图标）— 从列表中移除该市场；移除后该市场提供的插件从 发现 标签页消失，已安装的插件不受影响

注意事项 ​

若插件列表加载失败（提示未找到 Claude CLI 或 Codex CLI），请先在 常规设置 中配置正确的 CLI 路径，然后点击弹出提示中的 打开常规设置 跳转。

插件安装或状态变更后，运行中的对话会话将在 下次发送消息时 生效，无需重启应用。
