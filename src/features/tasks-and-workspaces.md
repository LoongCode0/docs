# 任务与工作区

LoongCode 用**工作区**（Workspace，对应你磁盘上的项目目录）和**任务**（Task，对应一次 AI 对话会话）两级结构来组织你的工作。本页介绍如何添加工作区、管理任务生命周期，以及置顶、搜索、归档等高效操作。

> 如果你还没开始第一个任务，建议先看[快速上手](/guide/quick-start)了解基础操作流程。

## 左侧任务栏布局

左侧任务栏从上到下的真实元素顺序为：

| 区域 | 说明 |
|------|------|
| **Logo / 折叠按钮** | 悬浮后可折叠侧栏（v0.7.0 起） |
| **搜索框** | 按关键词实时筛选任务 |
| **＋ 打开工作区** | 添加项目文件夹为工作区 |
| **＋ 新建任务** | 快速新建一个任务草稿 |
| **已置顶** | 已置顶任务集合，独立于分组折叠 |
| **任务大组头** | 含「任务 / 归档」标签与视图切换（分组列表 / 时间线） |
| **工作区树 + 任务列表** | 按工作区分组，展示各任务 |
| **底部栏** | 新手引导、移动端支持、打开设置 |

::: tip 快捷键
- `Ctrl+N`：新建任务草稿（任意界面可用）
- 悬浮 Logo 点击：折叠 / 展开左侧任务栏（v0.7.0）
:::

## 添加与管理工作区

### 添加工作区

1. 点击搜索框下方的**＋ 打开工作区**按钮。
2. 在弹出的文件夹选取对话框中选择本地项目目录，点击**确认**。
3. 如果该目录下有 Claude Code 历史会话，会弹出询问「**同步会话**」——点**导入**可将历史记录导入为任务；点**跳过**则跳过。

工作区添加成功后会出现在任务列表树中。

### 工作区操作

鼠标悬浮到工作区名称行，右侧出现三个快捷图标：

- **新建任务**：在该工作区下新建任务
- **查看文件**：将左侧栏切换为该工作区的**文件树**视图（文件树顶部的**返回任务**按钮可退回任务列表）
- **移除工作区**：从 LoongCode 中移除该工作区条目（**不影响磁盘上的实际文件**）

::: warning 移除工作区
移除工作区会删除该工作区下的所有任务记录，但磁盘上的代码与历史文件不受影响。
:::

### 拖拽调整工作区顺序

工作区分组在列表中的先后顺序完全由你手动决定（v0.9.8 起）：

- **按住工作区名称行拖动**即可调整顺序——移动超过几个像素才进入拖拽态（轻点仍是正常的展开 / 折叠），目标缝隙会显示一条**插入线**标示落点，松手落位，`Esc` 可取消本次拖拽。
- 顺序**持久化保存**，重启应用后保持不变；打开或切换某个工作区**不会**再改变它的位置。
- **新添加的工作区插入到列表最顶部**。
- 侧栏搜索框有内容（列表被过滤）时、以及归档视图下，拖拽暂时禁用；归档视图中工作区分组的先后顺序自动跟随主列表。

> 工作区**内部**的任务顺序仍按最近活动时间自动排列（见下文「任务排序与置顶」），不支持手动拖拽。

## 新建任务

### 草稿态机制

点击**＋ 新建任务**（或按 `Ctrl+N`）后，任务进入**草稿态**——此时尚未写入数据库，主区域内联显示配置选择器供你挑选工作区、模型、Git 分支或 worktree（Git worktree 是 Git 的独立检出目录，允许同时在多个分支上工作）。

**发送第一条消息后，任务才正式创建**，这样可以避免空任务堆积。

草稿会默认选中上次用过的工作区，少点一步。草稿选中的工作区被外部删除时，发送按钮会被禁用并给出提示。

### 草稿态下可用功能

即使任务还在草稿态，左侧栏的文件树，右侧面板的文件标签、审查、浏览器，以及终端均可正常使用，它们按工作区工作，不依赖任务是否已落盘。

## 任务状态

在默认的**分组列表**视图中，任务行的状态不再由单个圆点表达（v0.9.1 界面改版起），而是由三处元素共同传达：

- **左侧书签条**：任务行左缘的一条细竖条，只在「选中 / 运行中 / 提问中」时出现
- **标题明暗**：未读加粗醒目、已读变灰
- **行尾状态徽标**：任务行右侧的圆环 / 圆点等小徽标

任务状态仍归为五种：**空闲、运行中、成功、出错、提问中**。

### 各状态的外观

| 状态 | 任务行外观 |
|------|-----------|
| **空闲** / **成功** | 无特殊标记的普通行——「成功」只表示刚结束的一轮没有报错，任务随即回到等待输入的「空闲」状态 |
| **运行中** | 左侧出现**金色书签条**（当前选中的任务满亮，未选中的半透明）；行尾显示**橙色旋转圆环**与**已工作时长**计时（「已工作 X秒」，每秒跳动） |
| **出错** | 行尾显示**红色小圆点**（有未读时）；查看后变为**红色空心圆环** |
| **提问中** | 左侧书签条变为**蓝色**、整行带淡蓝底色，行尾显示**蓝色「?」圆形徽标**（脉动）——优先级最高，一眼可见 |

::: tip 运行中的计时也包含后台工作（v0.9.1 起）
「运行中」的判定与计时把**子智能体和后台任务**也计算在内——即使主回合已经结束，只要还有后台子智能体、后台命令或[动态工作流](/features/workflows)（v0.9.2 起）在跑，任务行就仍显示橙色圆环并继续累计「已工作」时长（常驻服务不计入）。
:::

> 当前选中的任务即使处于空闲，也会显示满亮的金色书签条，用于标记「你正在看的任务」。

### 未读提示

有没有未读消息改由**标题明暗 + 金色圆点**表达：

- **未读**：任务标题**加粗**显示，行尾出现一枚**金色小圆点**
- **已读**：切换到该任务、看过最新回复后，标题恢复灰色，金色圆点消失

「运行中」不受已读影响——只要任务还在工作，橙色圆环与计时就始终可见，方便一眼看出哪些任务正在后台干活。

### 其它徽标

除上述主状态徽标外，任务行右侧还可能出现：

| 徽标 | 含义 |
|------|------|
| **⏲** | 该任务挂有[定时执行](/features/scheduled-send)（可与其他徽标并存）；一次性定时刚触发且未读时显示**紫色小圆点** |
| **分支标签** | 绑定了 worktree 的任务显示其分支名小标签 |
| **⚡N** | 正有 N 个后台任务在运行；有后台完成未读时该徽标亮金加粗。详见[后台任务活动](/features/background-tasks) |

> 行尾主状态徽标同一时间只显示一个，优先级为：**提问中 > 运行中 > 定时触发未读 > 出错未读 > 普通未读 > 出错已读**。

::: tip 时间线视图仍是圆点
**时间线**视图（见下文「视图模式」）的任务行保留了简洁的**状态圆点**：颜色对应任务状态（如运行中琥珀色、出错红色），已读后变暗淡，悬浮时圆点让位给图钉置顶按钮。
:::

## 任务排序与置顶

### 按最近活动排序

任务列表与时间线分桶按**最近活动时间**排序——刚发过消息或刚结束一轮对话的任务会自动浮到前面（v0.4.2 起）。

### 置顶任务

置顶通过任务行上的**快捷按钮**完成（不在右键菜单里）：

1. 将鼠标悬浮到任务行上——行首会出现一个图钉形的**置顶任务**按钮。
2. 点击该按钮即完成置顶。

置顶后任务出现在**已置顶**分组，跨视图、不受「折叠全部」影响。已置顶的任务悬浮时该按钮变为**取消置顶**，再点一次即可恢复。

::: tip
置顶任务受自动归档保护，不会被自动归档。
:::

## 搜索任务

搜索框（占位提示「搜索任务…」）位于侧栏顶部 Logo 区域下方，输入关键词后任务列表实时筛选。

- 无匹配时显示「未找到匹配任务」
- 点击搜索框右侧的**清除搜索**按钮可清空关键词

::: tip 只搜标题不够？试试深度搜索（v0.9.8 起）
侧栏搜索框只按**任务标题**过滤。想检索**会话内容**（用户消息、AI 回复、思考过程），点击搜索框右侧的**深度搜索**按钮或按 `Ctrl+Shift+F`——已输入的关键词会自动带入。详见[深度搜索](/features/deep-search)。
:::

## 归档任务

### 手动归档

右键点击任务行，选择**归档**即可（任务行悬浮时行尾的**任务操作**下拉菜单中同样可以归档）。

归档后任务从主列表移出，进入归档视图，主列表更加清爽。

取消归档：进入归档视图后，右键任务选**取消归档**，任务的活动时间会被重置，回到主列表最上方，方便立即继续。

### 从对话区归档

除了在侧栏任务行右键，也可以直接在**对话区的空白处**右键——只要该任务未归档且完全空闲（不在运行中），会弹出一个只含两项的精简菜单：

- **归档并关闭** — 归档当前任务，同时关闭它所在的分栏
- **归档并新建** — 归档当前任务，并在原位置打开一个新草稿；新草稿**继承被归档任务的工作区与 CLI/模型/思考深度/权限配置**，无需重新选择

任务运行中、已归档，或右键命中了文本选区 / 链接 / 文件等其他目标时，弹出的是对应的常规菜单（复制、打开链接等），不会出现这两项归档快捷操作。

### 自动归档

应用**启动时**与**每小时**各跑一次自动归档，按「最近活动时间」超过阈值自动将久未触碰的任务归档（v0.4.3 起）。

**保护规则**：运行中任务和已置顶任务不会被自动归档。

**调整阈值**：在**设置 → 常规**中可开关自动归档，并设置阈值数值与单位（小时 / 天 / 月），默认开启、7 天。

### 查看归档视图

1. 在任务大组头右侧点击**查看归档**。
2. 进入归档视图后，任务按归档时间降序排列。
3. 归档视图下自动隐藏置顶区和新建入口，没有归档任务时显示「暂无归档任务」。
4. 点击**退出归档**返回主任务列表。

## 任务右键菜单

右键任务行可弹出操作菜单，仅包含以下三类操作：

| 菜单项 | 说明 |
|------|------|
| **重命名** | 仅未归档任务显示 |
| **归档** / **取消归档** | 按任务当前归档状态二选一 |
| **删除** | 危险操作，需二次确认 |

置顶不在右键菜单中——它是任务行悬浮时行首出现的图钉快捷按钮，见上文「置顶任务」。

::: tip
任务行悬浮时行尾还会出现**任务操作**下拉菜单，它提供与右键菜单相同的操作。
:::

::: danger 删除任务
删除操作不可恢复，请确认后再继续。删除任务仅删除 LoongCode 内的对话记录，不影响磁盘上的项目代码。
:::

## 视图模式

任务大组头提供两种视图切换：

- **分组列表**：按工作区分组展示，可展开 / 折叠各工作区
- **时间线**：按时间桶（今天、昨天、本周……）排列，跨工作区浏览

::: tip 选中任务自动定位（v0.8.0 起）
无论当前处于哪种视图，当你（在分屏中切换分栏、或通过机器人 / 移动端等方式）选中一个任务时，左侧栏会**自动展开并滚动定位到它所在的分组**——已置顶任务定位到「已置顶」组，其余按当前视图定位到对应的工作区分组或时间线日期桶，省去手动翻找折叠分组的麻烦。
:::
