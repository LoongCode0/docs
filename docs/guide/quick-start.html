快速上手 ​

本页按照零基础用户第一次打开 LoongCode 的时间线，一步一步带你完成第一次 AI 对话。如果你还没有安装 CLI 或配置账号，请先阅读前置依赖。

首次启动：跟着新手引导走 ​

第一次打开 LoongCode 时，界面中央会自动弹出新手引导。引导共分 5 章，每章聚焦一个主题：

章节内容
起步欢迎 → 打开工作区 → 新建任务 → 设置 claude（即配置 Claude CLI，界面原文如此）→ 发送首条消息
看代码目录树 → 查看文件 → 编辑文件
进阶选项模型 → 思考深度 → 执行授权 → 浏览器标签页
Git分支 → 审查
设置常规 → 模型供应商 → 插件 → 技能 → MCP 服务器 → 使用统计

引导中带实操闸门的步骤需要你完成对应动作才会自动跳到下一步；其他步骤点下一步即可前进。你也可以随时点跳过此步或退出引导。

引导结束后，可在设置 → 常规 → 使用引导处按章重看，或点从头重看重新体验完整流程。

没有引导怎么办？

如果引导弹窗被意外关闭，打开设置 → 常规，找到使用引导区域，点击从头重看即可重新开启。

第 1 步：打开工作区 ​

工作区就是你要让 AI 帮你处理的项目文件夹。打开工作区之后，AI 才能读取其中的代码和文件。

下面两条路径二选一即可。初次尝试推荐路径 A，后续步骤也以它为例；想直接用自己的项目就走路径 B，之后的操作完全相同。

路径 A：使用示例工作区（推荐初次尝试）

  1. 在新手引导「打开工作区」步骤中，点击打开示例工作区按钮。
  2. LoongCode 会自动创建并打开一个内含 README、脚本和笔记的示例项目文件夹。
  3. 左侧任务栏中出现示例工作区的名称，说明已成功打开。

路径 B：打开自己的项目

  1. 点击左侧任务栏中搜索框下方的**＋ 打开工作区**按钮。
  2. 在文件选择对话框中找到你的项目文件夹并确认。

第 2 步：新建任务 ​

任务是一次独立的 AI 会话——每个任务拥有自己的对话记录、配置和上下文，互不干扰。

  1. 点击左侧任务栏中搜索框下方的**＋ 新建任务**按钮。
  2. 中央区域切换为草稿态——草稿是任务还未正式开始时的预备状态，首条消息发送后任务才正式创建。

另一个入口

把鼠标移到左侧任务栏中的工作区名称上，名称右侧会出现新建任务图标，点击它同样可以在该工作区下新建任务。

草稿会记住上次配置

新建任务时，CLI、模型、思考深度、执行授权等选项默认继承上一次任务的设置，无需每次重新调整。

第 3 步：配置任务（可全部保持默认） ​

草稿态界面底部的选择器条显示当前任务的所有配置项。对于初次使用，默认值即可满足大多数需求。

配置项说明新手建议
CLI选择驱动引擎：Claude CLI（Anthropic 出品）、Codex CLI（OpenAI 出品）或 OpenCode CLI（开源，v0.9.0 起）保持 Claude CLI
模型选择具体的 AI 模型保持默认
思考深度控制 AI 的推理深度，越深越细致但更慢；Claude 选项：low / medium / high / xhigh / max / ultracode保持默认（出厂默认为 max），嫌慢可调低至 high
执行授权控制 AI 执行文件操作和命令的方式；Claude 选项：default / plan / acceptEdits / auto / dontAsk / bypassPermissions建议改为 default（每步操作先征求确认；出厂默认为 bypassPermissions，即完全自主执行）
工作区当前任务关联的项目文件夹无需修改
分支当前 Git 分支（仅对 Git 仓库显示）无需修改

首次使用需要先配置 API 密钥

如果你还没有配置过账号或 API 密钥，AI 将无法正常应答。请先前往设置 → 模型供应商完成配置，再回来发消息。

第 4 步：发送第一条消息 ​

输入框的占位提示为：「向 Claude 提问，输入 @ 添加文件，/ 使用命令，$ 使用技能」

  1. 点击中央输入框，光标出现在框内。
  2. 输入你的问题。对于示例工作区，可以试试：「介绍一下这个工作区里有什么」
  3. 按 Enter 键（或点击发送按钮）发送消息。

消息发出后，草稿态结束，任务正式创建，AI 开始处理你的请求。

输入框触发器

  • 输入 @ — 从工作区文件列表选择文件，添加到对话上下文
  • 输入 / — 打开命令面板，使用内置命令（如 /compact 压缩对话上下文）、插件命令和技能

第 5 步：看懂 AI 的回复 ​

回复的几种形态 ​

  • 流式文字：AI 的文字回答实时逐字显示，Markdown 格式自动渲染。
  • 思考过程（折叠块）：当思考深度高于 low 时，AI 会先输出内部推理过程，默认折叠显示为「思考过程」，点击可展开查看。
  • 工具调用时间线：AI 执行文件操作（读取文件、写入文件、执行命令等）时，每次调用在时间线上占一行，显示状态符、工具名与结果摘要；点击该行可展开详情，修改代码时会显示 Diff（代码差异）。
  • 提问卡片：当执行授权设为 default 或 plan 时，AI 需要你确认才能继续操作——卡片上会出现允许 / 拒绝等按钮。

任务状态 ​

左侧任务栏中，任务行通过左侧书签条与行尾徽标表达任务状态（共五种：空闲、运行中、成功、出错、提问中）：

状态任务行外观
空闲 / 成功无特殊标记的普通行
运行中左侧金色书签条；行尾橙色旋转圆环 + 「已工作」计时
出错行尾红色小圆点（未读）或红色空心圆环（已读）
提问中左侧蓝色书签条；行尾蓝色「?」圆形徽标

有没有未读消息则看标题明暗：有未读的新消息时任务标题加粗、行尾出现金色小圆点；看过后标题恢复灰色、圆点消失。

几点说明

  • 「运行中」不受已读影响：只要任务还在跑，橙色圆环与计时就始终可见。
  • 各状态外观的完整说明见任务与工作区。
  • 「提问中」时，输入框占位提示会变为「请在上方卡片作答;或在此草拟下一条消息…」（界面原文如此）。

下一步 ​

完成第一次对话后，建议了解主界面的完整布局——前往界面导览了解每个区域的功能，或直接进入任务与工作区学习更多使用技巧。
