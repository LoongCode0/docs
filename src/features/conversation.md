# 对话面板详解

对话面板是你与 AI 交互的核心区域。除了基本的消息收发，它还提供流式输出、思考过程展示、工具调用卡片、上下文管理等丰富功能。本页深入介绍这些功能的用法与细节。

> 相关基础：[界面导览](/guide/interface-overview)、[快速上手](/guide/quick-start)

---

## 流式输出与运行时长

AI 的回复是**实时流式渲染**的——字符一边生成一边出现，Markdown 格式同步渲染，不需要等待整条回复完成。

**运行时长指示**（v0.7.0 起）：

- **运行中**：停止按钮左侧实时显示「已工作 X 秒 / X 分 X 秒」（`chat.elapsed.seconds` / `chat.elapsed.minutesSeconds`），让你随时掌握本轮已跑多久。
- **回合结束**：在该轮对话底部显示本回合总耗时（不足 1 秒不显示）。

## Markdown 渲染

AI 回复中的 Markdown 语法会自动格式化：

- **标题、粗体、斜体、列表**：自动渲染为对应样式
- **代码块**：语法高亮显示，带语言标注（v0.5.x 起支持高亮）
- **表格**：渲染为带边框的表格
- **链接**：点击后在内嵌浏览器**就地打开**（v0.3.0 起）；按住修饰键或点击中键可逃生到系统浏览器

---

## 思考过程块

当 AI 使用扩展思考（Extended Thinking）时，内部推理过程会以**思考块**的形式显示在正文之前（v0.3.1 起）。

- **流式时**：思考块默认展开，实时显示推理内容
- **完成后**：自动折叠，标题显示「**思考过程**」（`chat.thinking.label`）与持续时长（`chat.thinking.duration`：「持续了 X 秒」）
- **手动控制**：点击思考块标题可展开或收起（`chat.toolCard.expand` / `chat.toolCard.collapse`）

::: tip
思考内容的可见性取决于模型供应商——部分供应商仅返回签名而无正文，此时只显示思考状态。
:::

## 工具调用卡片

AI 调用工具时，对话区会显示对应的**工具卡片**。不同工具显示不同样式：

### 读文件 / 写文件

文件操作以卡片形式展示，写文件和编辑操作额外显示 **Diff 视图**（新旧代码对比，行级红绿高亮）。

### 编辑 Diff

代码修改以 Diff 卡片内联呈现，让你一眼看清每处改动。

### 网络搜索 / 其他工具

搜索、命令执行等工具同样以卡片展示，显示工具名、运行状态（`chat.toolCard.running`：「运行中」）和结果摘要。

### 子 Agent 子卡片

当 AI 派生子 Agent 处理子任务时，子 Agent 的完整对话会被路由进独立的**折叠子卡片**（v0.3.1 起，Codex 任务同样支持）。

- 子卡片标题显示子 Agent 运行状态：`chat.subAgent.running`（「子 Agent 运行中」）
- 子卡片可独立展开 / 收起
- 如需了解子 Agent 的详细配置，见[子智能体](/features/subagents)

## 权限请求卡片与计划审批卡片

### 权限请求

AI 请求执行敏感操作时，弹出**权限请求**卡片（`chat.perm.title`），包含：

- **允许**（`chat.perm.allow`）：批准本次操作
- **拒绝**（`chat.perm.deny`）：拒绝，并可选填理由（`chat.perm.denyReasonPlaceholder`：「（可选）告诉 Claude 为什么拒绝 / 希望怎么改」）
- 点拒绝后出现**确认拒绝**（`chat.perm.confirmDeny`）二次确认

### 计划审批

在 Plan 模式下，AI 会先展示计划等待审批（`chat.plan.title`：「待批准的计划」），你可以：

- **批准·自动接受编辑**（`chat.plan.approveAuto`）：批准计划，后续文件改动自动确认
- **批准·逐步确认**（`chat.plan.approveStep`）：批准计划，但每步文件改动仍需手动确认
- **拒绝·继续规划**（`chat.plan.denyReplanning`）：拒绝当前计划，让 AI 重新规划

## AskUserQuestion 提问卡片

当 AI 需要向你提问时（例如询问功能选项或确认细节），对话区显示**提问卡片**，输入框占位符同时变为：

> 「请在上方卡片作答;或在此草拟下一条消息…」（`composer.placeholderWaiting`，界面原文如此，分号为半角）

提问卡片支持三种交互形式：

- **单选**：点击某个选项直接提交
- **多选**：勾选多个选项后提交
- **其他…**（`chat.ask.other`）：选择「其他…」后在输入框自由填写自定义回答（占位符：`chat.answerInput.placeholder`「输入自定义回答,@ 引用文件…」）

提交后选项区域显示「已提交」（`chat.ask.submitted`）。

## 上下文用量指示器与手动压缩

上下文窗口（Context Window）是 AI 每次处理时能看到的最大内容量，提示词、工具调用和回复都占用这个空间（`composer.ctxUsageTip`）。

**用量指示器**位于输入框选择器条最右侧，显示「上下文用量」（`composer.ctxUsageLabel`）进度条，悬浮可看到详情：「上下文已用 X / 总量 Y」（`composer.ctxUsageDetail`）。

### 手动压缩

当用量偏高时，点击进度条旁的**压缩**（`composer.ctxCompact`）按钮，对话历史会被压缩摘要，释放上下文空间。压缩后指示器显示「**已压缩**」（`composer.ctxCompacted`），并提示「对话已压缩,下一轮对话后重新计算用量」（`composer.ctxCompactedTitle`）。

### 自动压缩

当对话触达上下文上限时，LoongCode 会自动压缩并续跑（v0.3.1 起，需在**设置 → 常规**中开启「自动压缩」，默认开启）。

::: tip
手动压缩（`/compact` 命令或点击压缩按钮）和自动压缩都会保留对话摘要，AI 不会「失忆」。
:::

## 中断当前回合（优雅停止）

运行中点击**停止**（`composer.stop`）按钮，等价于 `ESC`——AI 停在当前位置，**已生成内容被保留**，你可以接着追问或修改（v0.4.1 起）。

**智能区分行为**：

- 本轮**尚未开始生成**时：停止表现为「撤回」刚发出的消息
- 本轮**已开始生成**时：表现为「就地中断」，被打断的回合显示「已被你中断」标记

中断时若有挂起的 AskUserQuestion，会补「未答」留痕，历史回放与实时态一致。

## 编辑历史消息重跑

你可以编辑任意一条历史用户消息，LoongCode 会以修改后的内容重新运行（v0.2.6 起）。

### 操作步骤

1. 找到你想修改的用户消息气泡，点击右侧**编辑**按钮（或双击消息）。
2. 消息进入编辑态，占位符提示：「修改后回车发送，或点 X 取消…」（`chat.edit.placeholder`）。
3. 修改内容后按回车或点击发送，LoongCode 自动从该消息处**分叉会话**并重新运行。

### 同时回滚代码

编辑历史消息时，如果该消息发出前有代码快照，可以勾选**同时回滚代码**（`chat.edit.rollbackCode`），将工作区代码回滚到该消息发送之前的状态。

勾选后会显示内联 Diff 预览，展示将被回滚的文件变更。回滚前会自动存一份可恢复的安全快照（`chat.edit.rollbackDialogDesc`）。

![截图：历史消息编辑态，勾选「同时回滚代码」后内联显示 Diff 预览，右侧有确认回滚按钮](...)

::: warning 代码回滚影响范围
回滚会丢弃该消息此后所有的文件改动，包括你手动的改动。操作前请确认 Diff 预览。如果没有代码快照，按钮不可用（`chat.edit.rollbackUnavailableTitle`：「此消息无代码快照」）。
:::

详细的代码快照与恢复机制见[代码快照](/features/checkpoints)。

## 会话分叉

在任意一条 AI 回复底部，点击**分叉到新会话**（`chat.forkToNewSession`），可以将当前对话历史复制一份，在新任务中从那个节点继续探索不同方向，而不影响原有对话。

---

## 历史完整还原

关闭应用再重新打开后，所有任务的对话历史都会**完整还原**，包括：

- 用户消息与 AI 回复
- 思考过程块
- 工具调用卡片与结果
- 上下文用量

长会话采用渲染窗口化，首屏只渲染可视范围内的消息，上滑后增量加载更早的历史（v0.4.1 起），切换长会话不再卡顿。
