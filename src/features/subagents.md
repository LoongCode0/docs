# 子智能体

子智能体（Sub-agent）是由主 Agent 派生出来的专项 Agent——主 Agent 在处理复杂任务时，可以将某个子任务交给子智能体独立执行，子任务完成后结果汇总回主流程。

> 相关基础：[对话面板 → 子智能体卡片](/features/conversation#子智能体卡片)

::: tip 一个类比
可以把它想象成：AI 面对一个大工程时，自动雇了几位「专项小助手」分头干活——有人专门查资料、有人专门改代码——各自干完后向主 AI 汇报结果，主 AI 再汇总推进。
:::

## 对话中的子智能体卡片

### 这是什么

每当主 Agent 派生一个子智能体，对话区会出现一张**折叠卡片**，将子智能体的完整对话过程封装在其中。v0.9.1 界面改版起，这张卡片采用「头卡 → 工作报告 → 完整流水」的**三层渐进披露**结构——默认只给一行概览，需要多少细节就展开多少。

### 第一层：头卡

卡片折叠时只显示一行**头卡**：

- 左侧为 ◈ 标记 + 子智能体名称（未指定专项名时显示「子智能体」）+ 子任务摘要
- **运行中**：名称旁出现「子 Agent 运行中」脉动徽标；**后台（异步）**子智能体完成后显示「后台」徽标
- 右侧依次为**调用次数**（如「5 次调用」）、耗时，以及一枚**状态圆点**——绿色 = 成功、红色 = 出错、橙色（脉动）= 运行中——和展开箭头

若子智能体尚未产生任何输出，展开后会显示占位提示「等待子 Agent 输出…」。

### 第二层：工作报告

点击头卡（或按 Enter / Space）展开**工作报告**，从上到下依次是：

1. **任务简报**：主 Agent 派发给子智能体的任务说明
2. **回执**（仅后台异步子智能体有）：派发时的确认回执
3. **过程摘要**：一行「⚙ N 次调用」加各工具的调用次数统计（如 `Read×4 Grep×2`）；其下默认只保留**最近 2 个产出型工具**（写文件、编辑、命令等会改变工作区的调用）的卡片，查询型调用（读文件、搜索等）只计数不展示
4. **结果框**：永远排在报告末尾——成功为绿色左条的「✓ 结果」，出错为红色左条的「✗ 结果」，内容为子智能体的最终汇报

### 第三层：完整流水

想看全过程时，点击报告内的**展开全部**——子智能体的完整流水（思考、正文、全部工具调用）会以与主对话一致的排版逐条展开；点击**收起全部**回到精简报告。

::: tip 后台（异步）子智能体
主 Agent 也可以把子智能体放到**后台**异步执行（v0.9.0 起）。后台子智能体同样以这套卡片呈现，且**默认折叠**（底部后台状态条仍可见）、展开后内容实时流式出现；完成后主 Agent 通常会在主对话里给出一段友好的结果描述。详见[后台任务活动](/features/background-tasks)。
:::

## 设置 → 子智能体

### 这是什么

**设置 → 子智能体**页面是自定义子智能体的可视化管理中心，你可以在这里新建、编辑、启用/禁用、删除用户级和项目级子智能体，也可以查看插件提供的子智能体。

::: tip 按 CLI 家族切换
页面顶部有 **Claude | Codex | OpenCode** 切换栏，分别查看各 CLI 的子智能体（OpenCode 家族自 v0.9.0 起纳入）。**只有 Claude 家族支持在应用内新建 / 编辑 / 删除**；Codex 与 OpenCode 的子智能体为**只读**——本页仅列出它们并提供启用 / 禁用开关（OpenCode 的开关会改写对应文件的 frontmatter `disable`），不提供创建 / 编辑 / 删除入口，也不显示 Claude 家族才有的**打开目录**按钮，以免误改各 CLI 的目录。下文的新建 / 编辑 / 删除步骤均针对 Claude 家族。
:::

子智能体按作用域分为三类：

| 标签 | 含义 |
|------|------|
| **用户** | 跨项目可用，存储在用户配置目录 |
| **项目** | 仅当前工作区可用（需有活动工作区） |
| **Plugin**（界面原文） | 由插件提供，因此只读，需在插件侧修改 |

### 在哪里打开

**设置**（右上角齿轮图标）→ 左侧导航 → **子智能体**

### 新建子智能体

1. 点击右上角**新建子智能体**按钮，打开新建表单。
2. 填写以下字段：

   | 字段 | 说明 |
   |------|------|
   | **名称** | 只能包含字母、数字和连字符，长度 3–50 个字符 |
   | **颜色** | 子智能体标识颜色（可选） |
   | **作用域** | 用户 / 项目（选项取决于是否有活动工作区） |
   | **描述** | 简要说明该子智能体的用途 |
   | **系统提示词** | 指导子智能体行为的核心指令 |
   | **工具** | 允许该子智能体使用的工具分组；不选则不限制，继承全部工具 |
   | **模型** | 下拉选择，选项有 inherit、opus、sonnet、haiku；选 inherit 表示继承当前会话模型（选择器下方有提示「inherit 继承当前会话模型」） |

3. 填写完成后点击**保存**，页面自动返回列表，新建的子智能体出现在列表中并提示「已新建 {name}」。

### 编辑子智能体

1. 在列表中找到目标子智能体，点击进入详情。
2. 详情页显示名称、描述、系统提示词、作用域、状态、模型、工具、文件路径等所有字段。
3. 点击**编辑**进入编辑表单，修改后保存即可，提示「已保存 {name}」。

### 启用 / 禁用

在列表中切换每条子智能体右侧的开关，状态立即变为**已启用**或**已禁用**。被禁用的子智能体不会被主 Agent 调用。

### 删除子智能体

在详情页点击删除，弹出确认对话框：

> 「确定删除「{name}」？文件将移入回收站。」

确认后文件移入系统回收站，列表刷新并提示「已删除 {name}」。

::: tip 项目级子智能体需有活动工作区
新建「项目」作用域的子智能体时，需要先在主界面打开一个工作区，否则选项置灰并提示「需有活动工作区」。
:::
