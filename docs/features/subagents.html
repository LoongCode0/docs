子智能体 ​

子智能体（Sub-agent）是由主 Agent 派生出来的专项 Agent——主 Agent 在处理复杂任务时，可以将某个子任务交给子智能体独立执行，子任务完成后结果汇总回主流程。

相关基础：对话面板 → 子智能体卡片

一个类比

可以把它想象成：AI 面对一个大工程时，自动雇了几位「专项小助手」分头干活——有人专门查资料、有人专门改代码——各自干完后向主 AI 汇报结果，主 AI 再汇总推进。

对话中的子智能体卡片 ​

这是什么 ​

每当主 Agent 派生一个子智能体，对话区会出现一张折叠卡片，将子智能体的完整对话过程封装在其中。当前版本的卡片采用「头卡 → 展开体」的两层渐进披露结构——默认只给一行概览，点开即看到该子智能体的完整对话，不再需要额外的「展开全部」二次操作。

早期版本是三层结构

v0.9.1～v0.9.6 期间卡片曾是「头卡 → 精简工作报告 → 完整流水」三层，展开后先看到裁剪过的「最近 2 个产出型工具」摘要，须再点一次「展开全部」才能看到完整过程，且末尾固定有一个独立的「结果」框。当前版本已简化为两层：点开头卡即直接看到完整回合列表，不再有单独的「结果框」——最后一条回复文字本身就是结论。

第一层：头卡 ​

卡片折叠时只显示一行头卡：

  • 左侧为 ◈ 标记 + 主题（优先取子智能体派发时携带的 description 简述；未携带时退化为任务提示前 40 字，仍取不到则显示「子智能体」）；主题右侧若使用了自定义专项子智能体（而非通用默认类型），额外显示一枚类型小标签
  • ⚡ 徽标：运行中（不论同步 / 异步）显示「⚡ 运行中」脉动描边徽标；异步（后台）子智能体运行结束后显示「⚡ 后台」描边徽标
  • 右侧依次为调用次数（如「5 次调用」）、耗时，以及一枚状态圆点——绿色 = 成功、红色 = 出错、橙色（脉动）= 运行中——和展开箭头

第二层：展开体 ​

点击头卡（或按 Enter / Space）展开，从上到下依次是：

  1. 统计小字：⚙ N 次调用 加各工具的调用次数统计（如 Read×4 Grep×2），有工具调用时才显示
  2. 指令气泡：主 Agent 派发给子智能体的任务说明，样式对齐用户消息气泡（右对齐、圆角），但用独立配色以示区分；正文过长时默认折叠，点击可展开全文
  3. 完整回合列表：按子智能体自己事件流的原始顺序，逐条渲染思考块、连续工具调用分组（与主对话工具调用时间线复用同一套渲染）、回复文本；若子智能体未把最终结论写进事件流，会在末尾自动补一条由回执文字合成的回复作为兜底结论——因此展开后始终能看到结论，不会因为没有专门的「结果框」而找不到

若子智能体尚未产生任何输出，展开后会显示占位提示「等待子 Agent 输出…」；异步子智能体已完成但还没有任何回复文字时，头卡下方会单独出现一行「✓ 后台子 Agent 已完成」的简短提示。

续聊时的指令气泡

主 Agent 通过 SendMessage 继续与某个已存在的子智能体对话时（例如追加澄清或下一步指示），这条续聊指令会作为同款指令气泡插入到回合列表的对应位置就地渲染，而不是被静默丢弃——展开卡片即可看到协调者在对话中途补充了什么。

后台（异步）子智能体

主 Agent 也可以把子智能体放到后台异步执行（v0.9.0 起）。后台子智能体同样以这套卡片呈现，且默认折叠（底部后台状态条仍可见）、展开后内容实时流式出现；完成后主 Agent 通常会在主对话里给出一段友好的结果描述。详见后台任务活动。

从 SendMessage 跳转到目标子智能体 ​

主 Agent 通过 SendMessage 工具给某个子智能体发消息时（例如续聊或让它继续下一步），对话主时间线上对应的 SendMessage 调用行会在能解析出目标时显示一枚跳转到该agent入口（箭头图标 + 文字）。点击后：

  • 若目标子智能体卡片当前处于折叠状态，会自动展开；若目标嵌套在更深层的子智能体卡片内部（多层嵌套派生），沿途所有祖先卡片会一并自动展开，不需要逐层手动点开
  • 展开完成后自动滚动定位到目标卡片并短暂高亮闪烁，方便在长对话里快速找到该子智能体

找不到可解析目标时（例如目标子智能体是在本次会话之外派生、或本次未发生过它的任何实时事件）该入口不显示，不会出现「点了却提示找不到」的情况。

设置 → 子智能体 ​

这是什么 ​

设置 → 子智能体页面是自定义子智能体的可视化管理中心，你可以在这里新建、编辑、启用/禁用、删除用户级和项目级子智能体，也可以查看插件提供的子智能体。

按 CLI 家族切换

页面顶部有 Claude | Codex | OpenCode 切换栏，分别查看各 CLI 的子智能体（OpenCode 家族自 v0.9.0 起纳入）。只有 Claude 家族支持在应用内新建 / 编辑 / 删除；Codex 与 OpenCode 的子智能体为只读——本页仅列出它们并提供启用 / 禁用开关（OpenCode 的开关会改写对应文件的 frontmatter disable），不提供创建 / 编辑 / 删除入口，也不显示 Claude 家族才有的打开目录按钮，以免误改各 CLI 的目录。下文的新建 / 编辑 / 删除步骤均针对 Claude 家族。

子智能体按作用域分为三类：

标签含义
用户跨项目可用，存储在用户配置目录
项目仅当前工作区可用（需有活动工作区）
Plugin（界面原文）由插件提供，因此只读，需在插件侧修改

在哪里打开 ​

设置（右上角齿轮图标）→ 左侧导航 → 子智能体

新建子智能体 ​

  1. 点击右上角新建子智能体按钮，打开新建表单。

  2. 填写以下字段：

    字段说明
    名称只能包含字母、数字和连字符，长度 3–50 个字符
    颜色子智能体标识颜色（可选）
    作用域用户 / 项目（选项取决于是否有活动工作区）
    描述简要说明该子智能体的用途
    系统提示词指导子智能体行为的核心指令
    工具允许该子智能体使用的工具分组；不选则不限制，继承全部工具
    模型下拉选择，选项有 inherit、opus、sonnet、haiku；选 inherit 表示继承当前会话模型（选择器下方有提示「inherit 继承当前会话模型」）

  3. 填写完成后点击保存，页面自动返回列表，新建的子智能体出现在列表中并提示「已新建 {name}」。

编辑子智能体 ​

  1. 在列表中找到目标子智能体，点击进入详情。
  2. 详情页显示名称、描述、系统提示词、作用域、状态、模型、工具、文件路径等所有字段。
  3. 点击编辑进入编辑表单，修改后保存即可，提示「已保存 {name}」。

启用 / 禁用 ​

在列表中切换每条子智能体右侧的开关，状态立即变为已启用或已禁用。被禁用的子智能体不会被主 Agent 调用。

删除子智能体 ​

在详情页点击删除，弹出确认对话框：

「确定删除「{name}」？文件将移入回收站。」

确认后文件移入系统回收站，列表刷新并提示「已删除 {name}」。

项目级子智能体需有活动工作区

新建「项目」作用域的子智能体时，需要先在主界面打开一个工作区，否则选项置灰并提示「需有活动工作区」。
