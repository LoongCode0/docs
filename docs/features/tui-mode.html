终端 TUI 模式 ​

偏好命令行交互？终端 TUI 模式（v0.9.8 起）可以把一个任务分栏从图形对话切换成 CLI 原生的终端交互界面（TUI，即 Claude Code / Codex / OpenCode 自己的命令行对话界面），并续接同一个会话——终端里能看到并继续 GUI 中的对话，切回对话视图后终端里新聊的内容也会回读进对话记录（按 CLI 支持程度，见下表）。

相关基础：多任务分屏、集成终端

与「集成终端」的区别

集成终端是工作区的 Shell 终端（跑构建、装依赖）；终端 TUI 模式则是把AI 对话本身换到 CLI 原生界面里继续，两者互不影响。

进入终端模式 ​

点击分栏标题栏右侧的切换到终端模式按钮（终端图标，在切分 / 关闭按钮旁）。分栏主区随即嵌入一个终端，按任务当前配置拉起对应 CLI 的交互界面并恢复当前会话；终端模式下同一按钮变为切换到对话模式（对话气泡图标）。

进入有两个前提，不满足时按钮禁用并以悬浮提示说明原因：

  • 任务空闲——「任务正在运行，结束后可进入终端」；
  • 任务已有会话（至少成功发过一条消息）——「先在对话里发送一条消息，才能进入终端」；草稿分栏不显示此按钮。

配置随任务注入 ​

进入终端时，任务当前的模型、思考深度、执行授权、模型供应商（接口地址 / 密钥）、网络代理与自定义环境变量会等价注入 CLI，终端行为与图形界面保持一致。配置在进入时刻冻结——想换模型或权限档，先结束终端会话、改完再进。

三家 CLI 的支持程度 ​

能力Claude CodeCodexOpenCode
按任务配置注入（模型 / 权限 / 供应商 / 代理）✅ 完整✅ 完整（启动时固定）✅ 配置共享
续接同一会话✅ 直接恢复✅ 恢复同一线程尽力而为（共享会话库）
终端轮次回读进对话记录✅ 支持❌ 不回放（模型上下文仍连续）尽力而为

Kimi Code 任务暂不支持终端 TUI 模式（v0.10.0）：Kimi 任务的分栏切换按钮会提示不可用，后续版本再行接入。

TIP

Codex / OpenCode 任务在终端里进行的轮次即使不出现在对话记录里，模型的上下文也是连续的——回到对话视图接着聊，AI 记得终端里发生过什么。

终端模式期间的对话视图 ​

会话被终端模式持有期间（包括切走后的等待期），该任务的对话视图变为只读：

  • 顶部显示横幅「会话由终端模式持有」，切走等待自动结束时追加「（终端空闲后将自动结束）」；
  • 横幅提供两个按钮：进入终端（跳回终端视图）与立即结束（强制结束终端里的 CLI 进程，需二次确认，正在输出的内容会被中断）；
  • 输入框与选择器条禁用，发消息、分叉、编辑重跑、回滚、定时执行等入口一并互斥，避免两种形态同时驱动一个会话。

切走不强杀：空闲后自动收尾 ​

从终端切回对话、或把分栏换绑到其他任务时，终端进程不会被立刻杀掉：

  1. 若终端还在输出（AI 正在回复），进程保活继续跑；
  2. 终端输出持续安静约 10 秒（视为当前消息已处理完）后，进程自动结束，防止后台进程越积越多；
  3. 自动结束前切回终端视图，会直接复用原进程与画面，滚动缓冲不丢；
  4. 进程结束后（自动收尾或你在终端里 /exit 退出），对话历史自动刷新；停留在终端视图时会显示「终端会话已结束」覆盖层，可选择重新进入终端或返回对话。

切走前先处理完终端里的待确认项

「空闲」按终端输出是否安静判定。若终端正停在权限确认等待你输入，输出同样是安静的——切走约 10 秒后进程会被自动结束，该轮次作废。切走前请先处理完终端里的待确认操作（横幅会显示终端仍被持有）。

已知限制 ​

  • 使用中转 / 第三方供应商的 Codex 任务，终端界面里显示的模型名可能是内部占位名，实际生效模型以任务配置为准。
  • 个别情况下权限档位会为适配交互模式自动降级，横幅会注明「（权限档已降级适配交互模式）」。
  • 退出应用、删除任务或工作区时，对应的终端进程会被连带结束，不会残留。
