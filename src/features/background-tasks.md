# 后台任务活动

AI 在处理复杂工作时，会把一部分工作放到**后台**异步执行——例如启动一个开发服务器、跑一条耗时命令、派一个**异步子智能体**单独干活，或挂一个后台监视器（Monitor）。这些后台工作通常在主回合结束**之后**才陆续产出结果。

从此版本起（v0.9.0 起），LoongCode 会把这些后台活动**实时、就地**呈现出来：任务行与标题栏有指示灯告诉你「有几个后台任务在跑、有没有刚完成还没看的」，发起后台工作的工具卡底部会显示后台状态条，后台子智能体的对话可就地展开实时查看，命令 / 服务的输出可在右侧面板逐行查看。

> 相关基础：[对话面板 → 工具调用时间线](/features/conversation#工具调用时间线)、[子智能体](/features/subagents)、[任务与工作区 → 任务状态](/features/tasks-and-workspaces#任务状态)

::: tip 仅 Claude
后台任务活动指示与查看依赖 **Claude Code** 的后台执行能力（`run_in_background` 命令 / 异步子智能体 / Monitor），仅在使用 Claude CLI 的任务中可用。
:::

::: tip 与「动态工作流」的区别
本页讲的是 AI 把**单条命令或单个子智能体**放到后台跑；那种跨多个阶段、多个子智能体的成套流程是另一种能力，见[动态工作流](/features/workflows)，二者各自独立呈现、互不影响。
:::

## 什么会被当作后台任务

LoongCode 跟踪以下几类由 AI 发起的后台工作：

| 类型 | 来自 | 图标 | 说明 |
|------|------|:---:|------|
| **后台命令** | AI 以「后台方式」执行的 Bash 命令 | ⌨️ | 一条放到后台跑的命令（例如批量处理、长时间脚本） |
| **服务** | 启动后**常驻**的命令 | 🌐 | 形如 `dev` / `serve` / `start` / `watch` 的进程，或命令里带显式端口（如 `--port 5173`、`:3000`）会被识别为「服务」 |
| **异步子智能体** | AI 派出的后台子智能体 | 🤖 | 在后台独立工作的子 Agent，完成后把结果汇报回主对话 |
| **监视器（Monitor）** | 后台监视器工具 | ⌨️ | 持续观察事件并在触发时唤醒主 AI 反应 |

### 任务 vs 服务

后台项会被自动分成**任务**与**服务**两组：

- **任务**：会跑完并产生明确「完成」信号的后台工作（普通后台命令、异步子智能体、Monitor）。
- **服务**：启动后长期运行、不会自己「完成」的进程（开发服务器等）。服务会一直停在「运行中」，不会催你说它完成了。识别到端口时，还会在条目上显示一个 `🔌 :端口` 标记。

::: warning 关于「完成」的判定
只有收到 CLI 真实的**完成通知**才会把一个后台项标记为完成——分组只影响显示与计数，**绝不会**凭命令名臆测「已完成」。因此长期运行的服务会一直显示运行中，符合预期。
:::

## 指示灯：哪里能看到后台活动

### 任务行的运行指示与 ⚡N 徽标

后台活动会直接反映在侧栏任务行的状态上（v0.9.1 界面改版起，详见[任务与工作区 → 任务状态](/features/tasks-and-workspaces#任务状态)）：

- 任务**正有后台任务在运行**时——即使主回合已经结束——任务行仍算「运行中」：显示橙色旋转圆环与「已工作」计时，右侧同时出现一个 **⚡N** 徽标（N = 正在运行的后台任务个数）。
- 有后台项**完成且你还没看过**时，⚡N 徽标以**亮金加粗**提示；切换到 / 点开该任务即清除这种未读，徽标恢复半透明。
- 后台工作**全部跑完**后 ⚡ 徽标消失；后台完成会唤醒主对话汇报结果，任务行随之出现常规的未读提示（标题加粗 + 行尾金色圆点）。

::: tip ⚡N 只数「正在运行的任务」
`⚡N` 的 N **只统计正在运行的任务项**——不含服务、不含已经完成的项。服务单独归在「服务」组，不计入 N。
:::

### 标题栏的 ⚡N 指示灯

标题栏右侧也有一枚 **⚡N** 指示灯，它**跟随当前选中的任务**（与提交菜单、终端、文件面板的「跟随当前任务」语义一致）——显示的是**当前任务**正在运行的后台任务数；当前任务有后台完成未读时，图标会以高亮色（warn）提示。

- 没有选中任务（草稿态）、或当前任务没有任何后台活动时，该指示灯不显示。
- 点击它会打开**后台活动浮层**（见下文）。

---

## 后台活动浮层：分组速览

点击标题栏的 **⚡N** 指示灯，会在其下方弹出一个**后台活动浮层**，按「任务 / 服务」两组列出**当前任务**的所有后台项。每行复用工具卡的视觉风格，显示：

- 类型图标（⌨️ 命令 / 🤖 子智能体 / 🌐 服务）；
- 描述（命令串或子任务说明）；
- 运行状态（运行中 / 已完成 / 失败），服务还会显示 `🔌 :端口`；
- 右侧操作：命令 / Monitor / 服务可点**查看**打开输出日志（见下文）。

点击浮层外的空白处即可关闭。当前任务没有任何后台项时，浮层显示「暂无后台活动」。

## 就地查看：工具卡、子对话与日志面板

后台工作的呈现**完全融入现有的工具卡范式**，不另起新卡片。

### 工具卡底部的后台状态条

发起后台工作的那张工具卡（后台命令 / 子智能体）底部会追加一条**后台状态条**，随后台项状态实时变化：

| 状态 | 显示 |
|------|------|
| 运行中 | 脉冲圆点 +「后台运行中」 |
| 已完成 | ✅「后台完成」（命令 / 服务类还带「查看输出」入口） |
| 失败 | ⚠「后台失败」 |

### 后台子智能体：就地展开实时查看

后台**子智能体**的对话与普通[子智能体卡片](/features/conversation#子智能体卡片)完全一致——挂在发起它的工具卡内：

- 后台子智能体的工具卡与其它工具卡一样**默认折叠**（只显示摘要与底部的后台状态条），**点击标题行即可展开**，随后就能**实时**看到它的思考、回复与工具调用一边产生一边出现；
- 折叠态下底部的「后台运行中 → 后台完成」状态条仍然可见，因此即使不展开也不会丢失「正在运行 / 已完成」的提示；
- 完成后照常可折叠 / 展开；
- 子智能体的结果不会进右侧日志面板，而是就地在子卡里查看。

::: tip 为什么默认折叠
后台子智能体在运行中会持续产生大量流式内容，若默认展开会不断撑长对话、令滚动难以跟随到底部。因此其工具卡默认折叠、由你按需点开查看，运行可见性则由底部状态条保证。
:::

### 命令 / Monitor / 服务：右侧日志面板

对于**命令 / Monitor / 服务**类后台项（它们没有子对话），在工具卡的「查看输出」或浮层的「查看」处点击，会在**右侧面板**新开一个**日志标签**（ScrollText 图标），逐行展示该后台项的输出文件内容：

- 对应后台项**仍在运行**时，日志按约 2 秒自动重读尾部刷新；完成或关闭标签即停止刷新；
- 这是一个**按需、可关闭**的标签，不常驻——你不看时可以直接关掉。

::: tip 串联跳转
指示灯、浮层、工具卡三处通过同一个工具调用标识相互关联：你既能从浮层点「查看」直达日志，也能在对话里顺着发起命令的工具卡找到它的后台状态与输出。
:::

## 完成时的提醒方式

后台工作完成时，LoongCode **不会弹窗、也不会发出提示音**——提醒完全靠任务行的徽标来表达：

- 还有其他后台项**在跑**时：⚡N 徽标继续显示并更新计数，有完成未读时徽标**亮金加粗**；
- 后台工作**全部结束**后：完成会唤醒主对话汇报结果，任务行出现常规未读提示（标题加粗 + 行尾金色圆点）；
- **切换到 / 点开该任务**后，未读即清除，徽标与圆点退回常态。

这样你在专注于某个任务时，其他任务后台完成不会打断你；回头扫一眼任务列表，加粗的标题和金色圆点就会告诉你哪些后台工作有了结果。

## 主 AI 的友好完成描述

后台工作（尤其是子智能体）完成后，主 AI 通常会被**自动唤起**，针对结果在主对话里给出一段**友好的完成描述**（例如把子智能体的代码审查结论重整成一张表格）。LoongCode 会让这段自动产生的描述**实时出现在主对话窗口**，而不是只能等下次切走切回才从历史里看到。

此外，后台子智能体若主动给主对话发消息（SendMessage 到 main），其内容也会作为一条以「📨 后台 Agent：」开头的提示出现在主对话里。

::: tip
后台工作完成后，承载它的任务会**短暂转为「运行中」**（主 AI 起了一轮自动反应回合），随后正常回到空闲。这是预期行为——它保证了上面那段友好描述能实时流出。
:::

## 已知限制

- **应用重启后不再跟踪运行中的后台项**：后台任务的活体跟踪信息不会随会话历史落盘，重启 LoongCode 后，之前正在运行的后台项无法继续跟踪（已完成的对话内容仍可从历史还原）。
- **服务的停止**：浮层中服务条目的「停止」入口目前为占位（暂不可用），如需停止常驻服务，可在对应的命令 / 终端处结束进程。
